# 异常告知

> 真理来源之一。用户决策见 `用户决策/决策日志.md` 2026-05-31「严重异常不得静默」。
> 关联:`系统架构/06-app.md`(get_status / 事件)、`系统架构/05-memory` 与 `state.rs`(持久化降级)。

## 铁律

**任何严重影响主要功能的异常,禁止静默处理(吞错 / 默默降级)。** 必须第一时间给用户**醒目的视觉告警**,颜色要足够警告(红 / 橙)。告警在问题解除或用户确认前**不自动消失**。

反例(现状要改):`state.rs` 的 `store: Option<Store>`——DB 打不开时静默置 `None`,记忆全留内存、退出即丢,用户毫不知情。这正是要根治的静默降级。

## 严重度分级与视觉

| 级别 | 含义 | 视觉 | 例 |
|---|---|---|---|
| 致命(红) | 主功能不可用 / 有数据丢失风险 | 状态栏红灯常亮 + 弹窗(需用户知晓) | 持久化打不开→记忆纯内存;沙箱根目录失效 |
| 降级(橙) | 功能受损但可继续 | 状态栏橙灯 + 横幅,不弹窗 | 本地嵌入模型加载/下载失败→RAG 退化到词法/精确层;远程嵌入鉴权失败 |
| 提示(黄) | 轻微 / 可恢复 | 状态栏黄点 | 单次工具调用失败已重试 |

绿 = 一切正常。指示灯取最高未解除级别的颜色。

## 必须告警的异常清单(主功能相关)

1. **持久化不可用**:`Store::open` 失败 / 运行中写失败 → 记忆与设置不落盘。**致命(红)**。
2. **嵌入不可用**:本地 e5 加载或下载失败、且无远程槽兜底 → 第一层 RAG 失效(退化到精确层线性扫)。**降级(橙)**。
3. **LLM 连接/鉴权失败**:主模型不可用 → 不能对话。**致命(红)**。
4. **项目沙箱失效**:工作目录不可读写 → 文件工具全废。**致命(红)**。

## 机制(后端探测 → 事件 → 前端常驻 UI)

- 后端 `AppState` 持一份 health 状态(如 `storage_ok / embedder_ok / llm_ok / sandbox_ok` + 各自原因串)。
- 探测点在异常发生处写入 health 并 `emit("health-alert", { level, code, message })`;不再用 `let _ = ...` 吞掉影响主功能的错误。
- `get_status` / `get_control_state`(已有桩)扩展返回 health 真值,前端轮询兜底,事件即时推送。
- 前端:状态栏常驻指示灯(绿/黄/橙/红)取最高级;致命级弹 modal;点灯展开异常详情与处置建议。问题解除时后端 emit 恢复事件,灯转绿。
  - **常驻 + 连接态合并(用户决策 2026-06-02)**:指示灯**始终显示**(不再"绿就隐藏")。**健康异常优先**(红 fatal / 橙 degraded / 黄 notice,并显示文字命名问题);**健康正常时该灯反映连接态——绿=已连接且正常,蓝=未连接**(蓝中性,避免与降级橙/提示黄混淆)。点开:有异常→列异常+复制按钮;无异常→一行状态("已连接,正常" / "未连接,请先连接模型")。即"一个常驻状态点,靠颜色表达 错误(红)/未连接(蓝)/已连接正常(绿)"。

## 落点(增量)

- 后端:`state.rs` 把 `Store::open` 失败从静默 `None` 改为记录 health「致命:持久化不可用」并保留内存降级运行(不崩,但告警);`build_local_embedder`/`LocalE5Embedder` 加载失败上报「降级:嵌入不可用」。
- 事件:新增 `health-alert` 事件 + `get_health` 命令(或并入 `get_status`)。
- 前端:状态栏指示灯组件 + modal;监听 `health-alert`。详情弹窗右下角有**复制按钮**(风格匹配暗底弹窗),复制全部异常(含 `code` 错误码)到剪贴板,便于用户联网查询/喂给 AI 排查(用户决策 2026-06-02)。
  - **具体例子(交互)**:详情弹窗**点指示灯展开,点击展示区域之外的任意空白处即收起**——监听 `document` 的 click,点到"指示灯+弹窗"整组之外才关,点按钮或弹窗内部(含复制按钮)不误关,收起即摘监听(用户原话见 `用户决策/决策日志.md` 2026-06-02「点击任意空白处…就能关闭那个展开」)。
- 原则贯穿:此后任何"降级路径"(`Option::None` 兜底、`unwrap_or_default` 吞错)凡涉及主功能,必须同时上报 health,不得静默。
