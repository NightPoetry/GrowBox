# 做梦/睡眠期一样占用潜意识 LLM 跑检索

## 结论
睡眠期不是"无检索"的后台清扫。其中的**演练(推演)**本质就是检索预演——自问自答"用户若问 X,该检索到什么",照着把索引提前建好。所以做梦/睡眠期间,潜意识 LLM 照样在走检索路径。

## 纠正的误解
直觉容易以为"做梦只是后台扫碎片、不碰搜索"。错。
- **做梦**:读碎片间隙内容 → 潜意识 LLM 判断有无遗漏(已经在读原文判相关,是轻检索)。
- **演练/推演**(睡眠 = 做梦 + 推演 交替):**直接就是模拟检索**,必然调潜意识 LLM 走一遍完整检索。
两者都占用同一个潜意识 LLM。

## 补强哪条设计
补强 `设计/02-记忆检索` 维护节那句"对话检索、做梦、飞轮提炼共用一个潜意识 LLM,优先级 Agent > Sleep > 飞轮"。

正因为做梦/演练自己也要检索,它和前台 Agent 检索抢的就是**同一个**潜意识 LLM——竞争是真实的,不是理论上的。

## 对工程的硬约束
造做梦/睡眠**之前**,必须先有潜意识 LLM 的仲裁器(带优先级的调度槽:Agent > Sleep > 飞轮),否则一上线就 race。这条对应地基评审里的"洞 1"(共用潜意识 LLM 缺仲裁器),是做梦/睡眠的前置件,不能后补。
