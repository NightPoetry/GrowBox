# 00 — 架构总览

> 定稿基线。**除前端交互设计外,全部非 UI 结构重写**;旧 `crates/` 仅作"踩过的坑"参考,不作实现依据。有疑问自己实验(见 `实验记录/`)。

## 架构公理

> **一切能力皆"执行器",Agent 循环是唯一脊柱。**

"IDE 是 AI 的手脚"(`设计/00-交互层`)在代码上的兑现:每个 IDE 能力(文件、shell、建项目、弹窗、改设置、检索…)都是一个**统一形态的执行器**,自带四样——给 LLM 的定义、可逆/风险等级、可选的预填 UI、执行后吐一条经验。所有执行器经**一个注册表、一条分发路径**调用。根治旧代码"三套分发打架"。

## Crate 全景(6 个,10→6)

全部重写(前端交互设计除外)。下表"对应旧物"仅供定位踩过的坑,非实现依据。

| crate | 只管 | 不管 | 对应旧物(仅参考) |
|---|---|---|---|
| **core** | 共享类型:结论模型、经验、执行器 trait(含 ToolDef/ToolResult)、上下文窗口类型、Scope、项目/设置数据结构 | 任何具体逻辑 | 新建(并入旧 conclusion/context/project/settings 的类型) |
| **llm** | LLM 抽象、路由(三槽位)、工具调用解析、reasoning 解析、DLC 方言 | 何时调、调来干啥 | 旧 llm(重写;实测须加 reasoning/流式 index 拼接,见 `实验记录/00`) |
| **memory** | 检索:RAG + 精确层飞轮(指针/染色/缓存/二级索引/碎片) | 提炼/因果(归 learn) | 旧 retrieval(按 `设计/02` 重写) |
| **safety** | 沙箱、路径分级、风险评级、三种授权 | 执行工具本身 | 旧 auth(重写) |
| **learn** | 飞轮:收集→提炼→压缩→验证→泛化 + 调度 | 存储原文(归 memory) | 旧 learner(按 `设计/04` 重写) |
| **app** | Agent 循环脊柱 + 执行器实现 + 前端 | 上述各 crate 的内部 | 旧 gui(脊柱重建,前端交互复用) |

**砍掉**:`growbox` CLI;`conclusion`/`context`/`project`/`settings` 不单独成 crate(类型并入 core,逻辑并入 app)。**6 crate 定稿。**

## Agent 循环脊柱(= 全局数据流)

```
用户意图
  │
  ├─① 组装上下文 ── memory.retrieve(分层下沉) + core 窗口类型 + 系统提示词
  │                 接原理:02 分层检索
  ├─② 调 LLM ────── llm.chat_stream(沉默超时/截断检测/方言解析)
  │
  ├─③ 执行器分发 ── 注册表查执行器 → safety 过门(可逆?直接跑:交还裁决)→ 执行
  │                 接原理:00 控制反转/可逆性、03 越界即交还
  ├─④ 感知/验证 ── 收结果 → 自动纠错(最多3次)→ 闭环
  │                 接原理:01 全链路感知、自动化闭环
  └─⑤ 学习 ─────── learn 收集经验(异步,不挡主线)→ idle 时提炼/压缩/验证
                    接原理:01 飞轮地基、04 飞轮
  │
  └─→ 回到 ① 或 完成/失败/等待用户
```

## 接原理(每篇设计 → 落点)

| 设计原理 | 架构落点 |
|---|---|
| `00` 最小心智负担 / 控制反转 | 执行器自带预填 UI;Agent 循环把劳动占满中间 |
| `01` 感知 + 闭环自动化 | 循环 ④ 感知/验证;执行器吐经验 |
| `02` 分层检索 + 精确层飞轮 | memory crate;循环 ① |
| `04` 飞轮学习 | learn crate;循环 ⑤ + idle 调度 |
| `05` 工具系统 | 执行器 trait(core)+ 注册表/分发(app) |
| `03` 安全审查 | safety crate;循环 ③ 安全门 |

## 已知坑(旧架构的系统病 → 本次根治)
