# 二期 · 设计原理 · 写作规范

本文件夹是二期的**原理层**,回答"为什么、凭哪条公理"。格式与内核 `设计/` 一致:**原则 → 推论 → 案例**,
自上而下、由抽象到具体。

> 工程落地层是 `项目设计/`(回答"做什么、怎么落"),格式不同,见该文件夹 README。
> **铁律**:二期每条原则要么是内核某条公理在新场景的延伸,要么明确标注是二期新增的独立立场;不能与内核冲突。

## 三层(同内核)

1. **宏观原则(K 条,越少越好,通常 1–3)** — 本主题不可再约简的根本立场,每条一句话能说清。
2. **推论(若干)** — 从原则机械推导出的具体主张,每条必须回指它来自哪条原则(含内核公理)。
3. **案例(挂在每条推论下)** — 用真实场景把推论坐实。没有案例的推论是空话。

## 模板

````
# <主题>

## 原则
1. <一句话原则>(若延伸自内核,注明 ← 内核某公理)

## 推论
### 推论 1:<标题>(← 原则 X)
<可操作的具体主张>

**案例**:<真实场景把它坐实>
````
