# 实验记录 · 写作规范

本文件夹存放**真机实验的客观记录**。背景:旧 AI(DeepSeek V4)攒下的很多结论是错的(尤其关于如何使用模型),不可继承。**所有关于外部世界/模型行为的结论,必须由本工作区亲自实验验证,记录在此。**

> 与 `设计/`(为什么)、`系统架构/`(怎么连)不同,本层回答"**实测是什么**"。

## 格式(经验格式:客观、可复现、无评价)

每篇一个主题,每条记录四段:

1. **猜想/待验** — 要验证什么(常常是"旧 AI 的某条结论是否成立")。
2. **方法** — 怎么测(请求参数、变量),一次只改一个变量。
3. **观测** — 实际返回的客观事实(贴关键字段,不加判断)。
4. **结论** — 从观测推出什么 + 对开发的硬约束。证伪旧结论时明确标注。

## 规矩
- 只记客观事实,不用"好/坏/成功/失败"评价词。
- 可复现:贴出能重跑的请求要点。
- 旧结论被推翻 → 不删旧说法,标注"实测推翻"+ 新结论(呼应 `设计/02` 的演化原则)。
