# 飞轮学习

> 飞轮是系统的灵魂:把每次实践沉淀成可复用、可演化的认知。
> 接 `01-能力层` 的下游:感知产生反馈 → 反馈沉淀为经验 → 飞轮把经验压成理解。

## 原则

1. **认知压缩** —— 每次实践沉淀为结论;压缩越深,适用越广,但出错代价越大。经验 / 知识 / 理解不是三种类型,是**同一个结论模型**在压缩谱上的不同位置。
2. **自指** —— 同一套引擎既优化任务,也优化自己;飞轮转出的结论反过来改进飞轮怎么转。
3. **永不停转** —— idle 时飞轮按永久目标 P1→P2→P3 自驱。永久目标永不完成,每轮只推进一点。

## 推论

### 推论 1:五阶段闭环(← 原则 1)
收集 → 提炼 → 压缩 → 验证 → 泛化,首尾相接、循环上升。
- **收集**:每次操作的客观快照(环境/前提/改了什么/结果),不判断、不打标签、不推因果。
- **提炼**:多条同类经验 → 不变模式("X 和 Y 常相伴"),得知识。
- **压缩**:从知识反推**最少前提**、机械推出**最大推论**,每条推论 = 一个可验证猜想,得理解。
- **验证**:实验确认或证伪。三路径——可回滚→控制变量;不可回滚→观察累积;涉代码→改/建/跑/截图/日志(自验证闭环)。
- **泛化**:已有理解解决不了 → 放宽一条前提、跨域类比 → 产生新猜想 → 回到收集。
- **案例**:收集到"max_tokens=20 → flash 返回空参"经验(`实验记录/00`)→ 多条同类 → 提炼"token 不足则工具调用截断" → 压缩成理解"flash 是推理模型,reasoning 先吃 token,故预算须覆盖 reasoning" → 设计 `max_tokens=30` 实验证实 → 泛化猜想"凡推理模型皆需预留 reasoning 预算"(待验 pro)。

### 推论 2:结论——一个模型,信度算出来,只缩范围不删除(← 原则 1)
结论 = 前提 + 操作 + 预期后果 + 来源 + 信度。
- **压缩程度连续(0~1)**,飞轮自动调,不手工标;检索时高压缩(理解)排前(经过更多验证)。
- **信度是算出来的**:知识 = 支持 ÷(支持 + 矛盾);理解 = 已验证推论 ÷ 总推论。不是人贴的标签。
- **append-only**:被证伪不降权,而是找到漏掉的前提 → 建一个前提 +1、范围更窄的新版本,旧版标"被取代"、留进化史。**没有"推翻",只有"适用范围被更精确地界定"。**
- **案例**:理解"flash 给足 token 即正常"被反例(某模型不同)挑战 → 不删,而是加前提"当模型=deepseek-v4-flash" → 新版本范围缩窄,旧版指向它。

### 推论 3:元优化——引擎优化引擎(← 原则 2)
飞轮也收集"自己怎么干活"的**元经验**(检索命中率、耗时、压缩成功率)→ 提炼元知识 → 压缩元理解 → 自动调整检索策略 / 索引结构 / 目录组织 / 命名规则。对内对外同一套引擎,只是输入不同。
- **案例**:某域检索命中率持续偏低 → 元经验积累 → 元理解"该域标签粒度过粗" → 自动细化标签 → 命中率回升。

### 推论 4:三永久目标,idle 自驱(← 原则 3)
有用户任务时全力完成(L0 最高优先级)。idle 时按序推进:
- **P1 元优化**:让系统越来越快("磨刀")。
- **P2 探索冲动**:验证所有未验证猜想——"**不存在真正的噪音,只有暂未解释的信号**"。
- **P3 最大化自动化**:能自动的绝不手动,减少人工中间人。
- 用户可随时加 P4、P5……开放可扩展。**这三条是写入初始飞轮的种子目标。**
- **案例**:idle 时 P2 扫到一条 `UNKNOWN-FAILURE`(暂未解释的失败)→ 主动设计实验复现 → 找到隐藏前提 → 转成一条知识,缝隙补上。

## 与记忆 / 调度的边界
- 经验/结论的**存储与检索**归 memory;**提炼/压缩/验证/泛化**归 learn。
- 飞轮与对话检索、做梦共用一个潜意识 LLM,优先级 **Agent > Sleep > 飞轮**(见 `系统架构/00`)。
