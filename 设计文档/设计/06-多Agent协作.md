# 多 Agent 协作(前瞻方向)

> 承 `05-工具系统` 推论 7(触发式工具)。本篇是**前瞻方向**,非当前已建机制——诚实标注:GrowBox 现状是单机单 LLM + 它派生的纯软件子 Agent(后台任务)经端口/Supervisor 汇报;"多机多 LLM 协作"是这条原则**指向**的未来,逐步落地。
>
> **设计原话(用户 2026-06-03,逐字见 `用户决策/决策日志.md`)**:"让这些东西对于LLM来说就是触发式的""相当于是LLM派生了一个纯软件的Agent，并开放接口等待它汇报""这个接口也是LLM扫描了本机接口之后……找个最不会对系统产生影响的空闲接口开放""它知道这个接口被自己开放且占用……从而让它未来编写测试软件的时候有意的避开自己使用的端口"。"为未来的多机Agent协作打下基础。"

## 原则

1. **派生即协作的雏形** —— LLM 起一个后台任务,本质是**派生一个纯软件子 Agent**(脚本/服务/进程),并为它开一个接口等它汇报。母 Agent 与子 Agent 经接口通信,就是最小的"多 Agent 协作"。
2. **触发式接口 = Agent 间通信原语** —— "完成即通知母 Agent 的端口形式"(`05` 推论7)不只是为了省轮询,它本质上是一条 **Agent 间通信信道**。把它做对,单机的"母→子"汇报,天然可推广为"Agent↔Agent""机器↔机器"的沟通。
3. **自知自有资源,才能与他者共存** —— 每个 Agent 必须清楚**自己占了哪些资源**(端口/进程/文件),才能在与他者(其它子 Agent、未来的远端 Agent)并存时不踩踏、不误杀。自我感知(`内部状态感知层` / `感知告知-双受众`)是协作的前提。

## 推论

### 推论 1:触发式接口先行,协议后立(← 原则 1+2)
先让"母派生子、子经端口汇报"在单机跑顺(已由 `05` 推论7 + `PortOpen` 判据 + Supervisor 支撑);**通信内容的协议(消息格式、握手、鉴权)留待真有多 Agent 需求时再立**,不预设死。现状的"端口可连=完成"是最朴素的一位信号,够单机用。
- **案例**:`spawn_task done_when=port:N` —— 子任务起好开端口,GrowBox(母)连上即知"子就绪/完成"。这条信道未来可承载更丰富的消息。

### 推论 2:自有资源登记靠既有视图,不另造记忆(← 原则 3)
Agent 对"自己开了哪些端口/子 Agent"的自知,**由 `list_tasks` 提供**(每个后台任务的 tag/原命令/状态/端口都在),无需新建常驻记忆区(用户 2026-06-03 拍板)。提示词强制:用端口前必须 shell 检查占用 + 经 `list_tasks` 辨认自己的、区别对待(避开、别误杀)。
- **案例**:LLM 要选测试端口 → 先 `lsof`/`ss` 看系统占用 + `list_tasks` 看自己的监听端口 → 两者都避开 → 不会杀掉自己正在监听的子 Agent。

### 推论 3:跨机是"接口换地址",不是另起炉灶(← 原则 2)
单机的"母连子端口"换成"母连远端子的 host:port",通信模型不变。故**把单机触发式接口做扎实,就是为多机协作打地基**;跨机引入的新问题(网络鉴权、发现、容错)是增量,不推翻原模型。现状**未实现跨机**(osascript/本机端口仅单机),列为方向。

## 现状 vs 方向(诚实)

- **已有(单机)**:LLM 派生后台子 Agent(`spawn_task`);`DoneWhen` 触发式完成判据(exit/port/file/probe);完成 `notify` → Supervisor 唤醒母 Agent 感知;`list_tasks` 自知自有资源;退避兜底巡检(`reap_hung`)。
- **复用面(造物)**:AI 现造 UI(造物,见 `05` 推论 8 / `00` 推论 10)的"过程/自激活半"——定时、监听端口唤醒自己——**直接复用上面这套**(`spawn_task` + 端口自激活),不新增机制;造物新增的只有 UI 半。
- **方向(未建)**:子 Agent 主动 push 富消息(非仅"端口可连"一位信号)的回调机制(用户明确**不为省 CPU 提前建**,待真需求);跨机 Agent 发现/鉴权/容错;多 LLM 间的任务分派与结果汇聚协议。
- **不做**:为单机省 CPU 而提前造重型消息总线/回调框架——违"需求驱动、不预设死"(原则 `00-交互层`)。

> 一句话:**触发式工具(`05` 推论7)是手段,多 Agent 协作是它指向的远景。** 现在把单机的"派生—汇报—自知"做扎实,远景自然在其延长线上。
