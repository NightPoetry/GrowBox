# 自我负责(Self-Stewardship)

> 内核原理之一(2026-06-08 用户提出升格)。与交互/能力/记忆/安全/飞轮/工具/工作流并列。
> 一句话:**AI 要对自己的输入与输出负责**——它产出的东西自己核验,它消费的东西适配成自己最能执行的形态。
> 这是"越用越强 / 活的 IDE"的内核气质:模型为自己的产物兜底、并持续改造自己的工作条件。

## 原则

1. **模型对自己的输入与输出负责。**
   - **输出侧**:它给出的结论/动作,要由它自己**对真实证据复核**后才作数(不凭记忆背书)。
   - **输入侧**:喂给它的提示词,应改写成**它自己最能理解执行**的形态(而非人写的、对所有模型通用的一份)。

## 关键理论:decoder 自亲和(self-affinity)

纯 decoder 模型(GPT/deepseek 族)的**输入注意力的向量空间与模式**,与它**自己输出**的高度相似。
推论有二,正好对应输入与输出两侧:

- **它最能执行"自己写出来的话"** → 提示词应由当前模型自转译成自己的母语风格(输入侧,推论2)。
- **它最能识别"自己输出里的虚实"** → 但仅靠"再想一遍"会自我背书;必须把它拉回**外部真实证据**才打破幻觉(输出侧,推论1+3)。

## 推论

### 推论 1:输出侧 = 主动自检,grounded verification(← 原则 1)

任务收尾前,AI 拿自己即将给出的工作汇报,**重新读取相关文件/真实状态逐条核对**,改正证据不支持的说法、
标注无法验证的,再正式收口。已实现:`self_verify`(脊柱 finish 拦截 + `self_verify_prompt`,见 `计划` 与下方"落地")。

> **案例**:AI 改完一个工作流,自述"把白名单设成 [file_read, file_edit, workflow_return]、并加了一句
> '不要用外部工具'的提示"。核 redb 真身发现:白名单实际是 [file_read, ask_user] / [file_edit, file_read],
> 也没有那句提示。**它一次干多件事时过度声称了**。自检 = 让它收尾前重读工作流定义对照自述,自己抓出并改正。

### 推论 2:输入侧 = 提示词自转译(← 原则 1 + 自亲和)

用**当前主模型**把所有"喂给模型的提示词"(系统提示/工具说明/工作流引导/脊柱脚手架)按它自己的风格重写,
更清楚、更易执行;喂给**非主模型**(潜意识/判定/蒸馏)的那部分,由**那个模型自己**转译自己那份(谁转译谁)。
**v1 已实现(2026-06-08)**:`transpile.rs` 唯一取用 chokepoint `tr()` + 按消费模型分桶的覆盖层(默认关=逐字原文)+
角色 tag(Main/Subconscious,今天同一模型同桶、将来拆潜意识模型按 key 自动分桶,转译层不改)+ 保真校验。
覆盖范围 = 系统提示/工具 llm_desc/self_verify/judge·distill;as-built 见 `计划/提示词自转译.md` 文末。

> **案例**:同一句工具说明,人写的版本对模型 A 啰嗦、对模型 B 又太简;让 A 用自己的话重写它要执行的说明,
> A 读起来"像自己想的",执行更准。换模型即重转译——提示词随模型走,而非固定一份。

### 推论 3:必须 grounded,纯自省无效甚至有害(← 自亲和的反面)

"再反思一下/再想想"几乎不涨准确率:产生幻觉的模型,空想一遍仍会维护自己的说法(自我背书),有时反而强化错误。
**真正有效的是把模型拉回外部锚点**:

- 输出侧自检 → 锚在**真实文件/状态**(重读 file_read / lsp / code_outline / code_search / shell),不是重想。
- 输入侧自转译 → 锚在**模型自身**(由要执行它的那个模型来写),并对改写做**保真校验**(防语义漂移丢约束)。

> **案例**:抓出上面那次"过度声称"的,不是"让 AI 再想想对不对"(它会说"对"),而是**人工重读了 redb**。
> 自检把这一步自动化——强制重读真身,而非内省。

### 推论 4:可关、可还原、按需触发(← 与"一切可设"一致)

自我负责的两条都增加成本(自检 = 多一轮 LLM;自转译 = 每条提示词一次 LLM)。故都做成**开关 + 阈值/触发**,
默认开但圈定到"值得花"的场景,关掉即省 token;转译保留原文、可一键还原。绝不强制每次都跑。

> **案例**:`self_verify` 开关 + `self_verify_min_tools` 阈值——只有本次任务工具调用数达阈值(干了多件事、
> 易过度声称)才自检;纯问答/轻任务直接收口,不花这个钱。造物交互/后台 Supervisor 回合不自检。

## 落地(as-built / 规划)

- **推论 1(已实现)**:`core::Settings.self_verify` + `self_verify_min_tools`;`AgentConfig` 透传;
  脊柱 `agent/mod.rs` 在 finish 拦截(主链、达阈值、未自检过)→ 注入 `render::self_verify_prompt` → 续一轮、
  AI 重读后再 finish(至多一轮);动态指示器 `AgentEvent::Status`("正在核查 xxx",随对象变)。前端可开关 + 阈值。
- **推论 2(v1 已实现,2026-06-08)**:`transpile.rs`(`tr` chokepoint + 按消费模型分桶覆盖层 + 角色 tag +
  机械保真`fidelity_ok`+语义复核)+ `cmds/transpile.rs`(`transpile_prompts` 按需触发、不持锁跑 LLM)+
  `core::Settings.prompt_transpile`(默认关)+ AgentTab 开关/按钮。覆盖系统提示/工具说明/self_verify/judge·distill;
  as-built 见 `计划/提示词自转译.md` 文末。后续可扩:区头脚手架/node_guidance/独立潜意识模型槽。

## 与其它原理的关系

- 与 `04-飞轮学习`:飞轮是"把经验压成知识/理解";自我负责是"对每次产出/输入即时把关"。前者长期、后者当下,互补。
- 与 `内部状态感知`:感知是"看得见自己的失败";自我负责是"据此核验改正"——感知是眼,自我负责是手。
- 与 `02-记忆检索`:自检重读用的就是检索/文件能力;自转译的对象之一是注入提示词,需保持 byte-stable 前缀(缓存)。
