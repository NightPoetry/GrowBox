# 内部状态感知层 —— AI 能感知一切失败/内部状态

> 来源:用户决策 2026-06-01(逐字见 `用户决策/决策日志.md` 顶部"★核心架构★")。
> 本文件是该决策的设计落点说明。Opus 2026-06-01 本会话已实现。

## 原则(用户原话精炼)

1. **AI 能感知一切**:所有失败(LLM 调用失败、工具失败、持久化写失败等)AI 必须能感知。
2. **内部状态 = 特殊追加上下文,拼在最末**:它经常变动,放最后才不破坏前面的稳定 prompt 缓存前缀(与 P4 的 append-only 同理)。
3. **特殊句式标明"这是内部状态"**:系统自述,非用户输入,明确区分。
4. **感知的一切都能被索引**:内部报错也写进时间线 → 成为带时间戳、可被嵌入/检索的记忆。
5. **时间戳重建顺序**:索引项必须带时间戳;因检索会打乱位置顺序,AI 须据时间戳重建时间序(P4"按时间戳判先后不按位置"推广到一切被索引的东西)。

这是"严重异常不静默"铁律的另一半:
- 一级"异常告知"(`异常告知.md`)= 面向**用户**的红警灯(`health`)。
- 本层 = 面向 **AI 自身**的感知(内部状态块 + 时间线)。两者同源,失败同时走两路。

## 实现(as-built,Opus 2026-06-01)

### Memory(`growbox-memory/src/memory.rs`)
- `internal_events: VecDeque<InternalEvent>` —— 瞬态 RAM 环,容量 `INTERNAL_EVENTS_CAP=32`,超出丢最旧。`InternalEvent{at,kind,message}`。
- `perceive(kind, message)` —— 双路并行:
  1. 推入瞬态环(渲染进上下文最末块);
  2. `ingest_with_role(format!("[内部状态·{kind}] {message}"), "internal")` —— 落带时间戳的时间线节点,role=`internal` → 可嵌入(idle 补向量)、可检索。
- `render_internal_state() -> Option<String>` —— 渲染"内部状态"块:特殊句式分隔(`~~~ 内部状态 · INTERNAL STATE(系统自述,非用户输入)~~~`)+ 本块说明(明示随时变动、置最末、按时间戳判先后)+ 每条 `[时间 RFC3339 | kind] message`。无事件返回 None。
- `internal_event_count()` —— 瞬态环计数(观测/测试)。

### Agent 循环(`growbox-gui/src/agent.rs`)
- **置末**:每回合(retry 前)取一次 `memory.render_internal_state()`,在本回合请求里拼到 `messages` 之后的**最末**(`turn_messages = messages.clone(); push(internal)`)。`messages` 主体(system→工作区→ring→对话)保持稳定缓存前缀,内部块是唯一易变尾巴。
- **失败路由**:LLM 调用失败(`drive_one` Err)→ `perceive("LLM调用失败", e)`;工具非成功(`!result.ok`,含执行失败/被拒/需授权)→ `perceive("工具失败", "{name}: {content}")`。

### GUI 其它失败源
- `cmds.rs` `run_chat`:回合开始读 `store.write_fault()`,计数增长时 `perceive("持久化写失败", ...)` 一次(`AppState.perceived_write_faults` 去重)。
- `state.rs` `AppState::new`:开机 `Store::open` 失败 → `health.set(Fatal)` 同时 `memory.perceive("持久化打不开", ...)`。

## 诚实记一笔 / 已知限制
- 内部事件也进时间线 → 会同时出现在"最近记忆 ring"(时间线尾)与"内部状态块"。轻微重复,无害(ring 是历史视角,内部块是当前易变态)。
- 工具失败的 perceive 与喂回模型的 tool 结果消息并存:tool 结果是当回合瞬态(脚手架),perceive 让它持久化可检索 —— 这正是"感知的一切都能被索引"的要求。
- `internal` 角色是新角色;检索命中会把内部事件调进工作记忆区(期望行为:AI 可召回自己过去的内部状态)。
- 瞬态环有界(32),但时间线全留(可检索的历史不丢)。

## 测试
`growbox-memory`:`perceive_rings_renders_and_persists_to_timeline`(渲染含特殊句式+时间戳、进时间线)、`internal_events_ring_is_bounded_but_timeline_keeps_all`(环有界、时间线全留)。
