# 打包设计

> 真理来源之一。用户决策见 `用户决策/决策日志.md` 2026-05-31「打包:带模型包 + 不带模型包两变体」。
> 关联:`计划/embedding-service.md`(本地 e5 嵌入)、`计划/precision-layer.md`(磁盘原生)。

## 目标

本地优先的持久 agent,要在 macOS / Windows / Linux 三端干净分发。打包要解决两件事:
1. 前后端构建开关一致(已有:正式包无调试桥 / 测试包带 :19999 调试口)。
2. 本地嵌入模型(`multilingual-e5-small`,约 470MB)如何随包走。

本文件只管第 2 件叠加在现有脚本上的新维度:**模型是否随包**。

## 原则:同一套运行时解析,两种包只差"有没有预置文件"

运行时绝不写死"模型一定在某处"。统一一个解析顺序,带不带模型的包跑的是同一份代码:

```
解析 e5 模型文件(config.json / model.safetensors / tokenizer.json):
  1. resource_dir/models/multilingual-e5-small/   ← 带模型包预置在这里
  2. data_dir/models/multilingual-e5-small/        ← 上次下载的缓存
  3. 用 hf-hub 从 HuggingFace 下载到 data_dir/models/...,再走 2
```

- **带模型包**:构建时把权重塞进 `resource_dir/models/...`,步骤 1 命中,装好即用、离线、单次下载(随安装包)。
- **不带模型包**:不预置,步骤 1 落空 → 步骤 3 首次运行下载到 data_dir(需联网一次),之后步骤 2 命中。
- 两种包二进制完全相同,只是 bundle 资源里有没有那个 models 目录。代码无分支(除了"下载进度提示"对 lite 包有意义)。

这样也天然覆盖第三种情形:用户手动把权重丢进 data_dir/models 也能用(企业内网/离线机)。

## 两个打包变体(叠加在现有 official/test 之上)

现有两轴:正式(official)/ 测试(test)—— 控制调试桥与端口。
新增一轴:带模型(full)/ 不带模型(lite)—— 控制 resource 是否含权重。

落地方式(择一,倾向 A):
- **A. 给现有脚本加开关**:`scripts/build-official.sh --with-model` / 默认不带。脚本据此决定是否把 models 拷进 tauri resources,并改 `tauri.conf.json` 的 `bundle.resources`(或用单独的 conf 片段/环境变量注入,避免手改 json)。
- B. 新增 `scripts/build-official-full.sh` 薄封装,内部调 official 再叠加模型拷贝。

模型来源:构建机本地缓存目录(`~/.cache/huggingface` 或仓库 `assets/models/`,后者纳入 .gitignore 不入库,首次构建前用 hf-hub 拉一次)。

## 与现有构建/启动坑的衔接(沿用 CLAUDE.md 已知坑)

- `tauri.conf.json` 的 `beforeBuildCommand` 为空 → 改前端必须先手动 `npm run build`,否则旧 dist 入包。模型拷贝步骤要排在前端 build 之后、`tauri build` 之前。
- 二进制名 `growbox`;`scripts/launch.sh` 已含前端重建步骤。
- resource_dir 取法:Tauri 的 `app.path().resource_dir()`(已在 `state.rs` / `connect` 用它读 prompts),模型解析复用同一入口。

## 跨平台

- 模型文件是纯数据(safetensors + json),无平台差异;candle CPU 后端纯 Rust,三端通吃(这正是不用 ONNX-RT 的原因)。
- 体积:带模型包安装体积 +~470MB;lite 包小,但首次启动要联网下载,需在 UI 给下载进度与失败兜底(下载失败可暂退化到词法 embedder,提示用户)。

## as-built(full 变体落地 + 出包验证,Opus 2026-06-01)

落地走了"方式 A + 单独 conf 片段"(避免手改基础 json,基础包恒为 lite):
- **config 片段** `crates/growbox-gui/tauri.bundle-model.conf.json`:把 e5 三件套加进 `bundle.resources`(目标 `models/multilingual-e5-small/...`),`cargo tauri build --config` 与基础 conf 深合并。基础 `tauri.conf.json` 不含模型 → 默认即 lite。
- **脚本** `scripts/build-official.sh --with-model`:暂存模型(来源优先 `assets/models/<name>/`,回退 HF 本地缓存快照 `~/.cache/huggingface/.../models--intfloat--multilingual-e5-small/snapshots/*`)到 `crates/growbox-gui/models/<name>/`(gitignore,不入库)→ 追加 `--config tauri.bundle-model.conf.json`。省略 `--with-model` = lite。其余参数透传。坑:bash 3.2(macOS)+ `set -u` 空数组展开报 unbound,用 `${arr[@]+"${arr[@]}"}` 守护。
- **运行时解析链已通**(无分支):`cmds.rs connect` 设 `resource_dir`(`app.path().resource_dir()`)→ `state.rs build_local_embedder` 把 `resource_dir/models` 排在 `data_dir/models` 前 → `local_e5.rs resolved_dir` 在 `<root>/multilingual-e5-small/` 找齐三件即命中。full 包第一步命中(离线即用);lite 包落空走 hf-hub 下载到 data_dir。
- **出包实测(2026-06-01)**:`GrowBox_0.1.0_aarch64.dmg` = **273M**;`GrowBox.app` 483M,模型在 `Contents/Resources/models/multilingual-e5-small/`(465M,三件齐),`icon.icns` + 版本 0.1.0 均正确。**已验证=静态打包结果**;"打包态 app 启动→解析→candle 加载"端到端未在包内重跑(candle 加载这三文件本身已由 `live_e5` 证),属机械可靠的文件存在+路径拼接。
- **图标**:`icons/icon.svg`(矢量源,节点云+中央脊柱)→ qlmanage 栅格 `icon-1024.png` → `cargo tauri icon` 生成全套。**版本统一 0.1.0**(Cargo/tauri 权威 + App.tsx + package.json)。
