# 网页自反馈调试 —— AI 真操作 → 读结果 → 判对错 → 修(2026-06-16 用户提出)

> 用户要的很简单:**自反馈调试**。GrowBox 自测它做的网页时,现在只"看样式 / 枚举回调",不真操作 → 功能 bug(用户实测:**CSS 没问题了,但按钮跳转有问题**)漏掉。
> 要让它**像真人测试员一样真去操作再看结果**,形成闭环。别搞复杂(上一版我过度设计成"三大支柱",已删)。

## ★用户原话(逐字)★
> "加强一下测试的工作流或者 SKill 什么的,因为我发现它自己测试的时候 CSS 没问题了,但是按钮跳转有问题,它应该就像人类测试员一样,有计划有目的的点击各个按钮甚至提交表单等来测试功能是否正确。除非涉及真实的金融交易,此时会让用户进行测试(当然用户批准了它自动测试除外,但是用户没有任何历史记录说这个项目可以被批准操作实际金融时,它必须弹出第一次的提示)。"
>
> (纠偏)"我只是让你实现自反馈的调试,你这是设计了什么,说中文。"

## 就是这个意思(大白话)
- **自反馈闭环**:AI 测自己做的网页 = **真点按钮 / 真走跳转 / 真填表单提交 → 读回结果(跳对页没?报错没?404 没?状态变对没?)→ 判对错 → 不对就修**。有计划有目的地把功能点过一遍,像真人 QA。
- **现状缺口**:自测只枚举回调、不真操作(见 [[auto-debug-mode]] / 被造物 selftest 仍欠项)→ 所以按钮跳转坏了它察觉不到。补上"真操作 + 读结果"就行。

## 金融安全规矩(就一条,不是另起炉灶)
- 要**提交真实金融交易**时:**不自动做**。
- 这个项目**没有批准过自动金融操作的历史** → **第一次必须弹窗问用户**;用户批了就记住、以后放行;没批就交回用户自己测。

## 怎么做(复用工作流机制 —— 用户点透:这就是一个预编排工作流)
★金融安全控制 = **一个预编排的工作流**,跟内置 `command_safety`(给 shell 设安全闸)**同一套机制,不新造安全子系统**。★ 见 [[workflow-v2-stack-function]]/[[workflow-p1-implemented]];头号原则"流程即一等公民"([[phase2-design-complete]])。
- **手(工作流要调的工具)**:`crates/growbox-gui/src/web_debug_runtime.js`(注入被测网页、已能穿 CSP)从"枚举回调"升级到**真模拟操作 + 读结果回传**:派发真实 click、填 input value + 触发事件、submit form;操作后读 location/title/console 报错/DOM 变化,经哨兵导航(gxlasso 同款通道)回传主窗 AI。
- **流程 = 一个预编排工作流**:列交互点 → 逐个真操作(点/填/提交)→ 读结果核对 → pass/fail 报告。其中"金融提交"是个**被门控的节点**。
- **金融闸 = 工作流的强制流转**:流程走到"提交真实金融交易"节点(去往真实支付/下单、pay/checkout/charge 等)→ 强制拐去授权节点;按项目查授权,无历史则**首次弹窗问用户**;批过记住、以后该项目放行。复用 command_safety 的强制流转 + 节点收窄 + 按项目持久 + 决定脊柱授权([[decision-spine-unified]]),**零新子系统**。

## 落地(2026-06-16 0-OPUS44,★①②③ 全落地★)
- **① 手 = AI 可调工具**(commit 0dec5c5):`web_debug_drive` 执行器(家族二 round_trip),AI 能 click/fill/submit/scan/observe + 读回 url/title/报错/lastAction(选择器没匹配/点了没跳转/语法错都分得清)。
- **② 脑 = QA 技能**(commit 0dec5c5):`web-qa-self-feedback` 种子,指导有计划地 scan→逐个真操作→observe 核对→不对就改源码复测。
- **③ 金融闸 = `financial_action_gate` 预编排工作流**(commit 43f3adc,B 软闸):金融提交前栈调用它读 `{authorized}`,true 才提交;授权节点 tools 为空(只剩 ask_user/workflow_return)→ 结构性无提交能力。**为什么不硬拦:派发层机械判不准"是不是金融"(中文/图标按钮漏、查看支付记录误拦、点击式付款连表单都不是)→ 金融判断交给人在首次授权拍板,安全偏置=拿不准就问。** 详见决策日志 2026-06-16。
