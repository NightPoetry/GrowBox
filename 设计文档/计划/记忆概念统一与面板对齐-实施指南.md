## 1. 必须对齐的统一心智模型(用户原话提炼)
**(A) 一个存放区 + 一个索引区 —— 临时记忆只有一个存放区。**
- **存放区(临时记忆,唯一)** = 工作记忆区 = `growbox-memory/src/context.rs` 的 `ContextWindow.resident` = 面板「队列占用」。**检索到的内容(不论 RAG 还是 L2 找到的)都 `page_in` 进这唯一一个区** —— 这就是 AI"怎么记住"的。
- **索引区** = 定位"该把哪些记忆调入存放区"的指针/页表。**RAG(向量 ANN)和 L2(LLM 顺序扫 + 指针图导航)只是两种"索引手段",不是两个缓存、更不是两个存储。**
- **两态铁律**:存放区 append-only、信息只"在/移除"两态、**绝不随热度重排**(排队/LRU 动的是索引区的指针元数据,不是存放区内容)→ 上下文字节稳定 → 命中 LLM prompt KV 缓存(省钱)。

**(B) 指针生命周期 + 纯净性。**
- 指针是"被索引"的副产品(被索引就产生指针)。
- **一次指针** = 顺序扫的产物,底下路径已(直/间接)扫过 → **无碎片**;对应染色≈深绿。
- **二次指针** = 跨"尚未扫的中段"的活动锚(可能漏,染色≈浅绿):换出 L2 在换出位置留标记、再被索引到移回 L2、位置始终可追(防崩溃)、**碎片整理(做梦)后升级成一次指针**。
- **碎片 = 用指针/锚"跳过中段"直达目标时,被跳过的未扫中段**(欠债);做梦扫它还债 + 把二次升一次。
- **LLM 指针**(L2 顺序扫)→ 有碎片回收机制。**RAG 指针**(向量跳):GrowBox 用 **ANN(不顺序扫)** → 应作**假指针**:统一进队列/缓存吃置换,但**不留序列二级锚、不进碎片系统**(ANN 无扫描 gap,没债可还)。

## 2. 现状(审计结论:基本落地,差在"概念没显式化/命名没统一")
✅ 已对的:
- RAG = ANN(`index.rs` 内存 `HnswIndex` / 磁盘 `ArroyIndex`)。
- LLM/L2 侧完整:`secondary.rs` 二级锚严格 K/2K 规则;`retrieval.rs:503` 跳过中段即 `fragments.record` 记碎片;做梦回收。**LLM 指针有碎片回收。**
- RAG 命中:`retrieval.rs` retrieve()(约 line 88-95)只 `touch+persist`,再由 `assemble_context`(line 602-611)`page_in` 进存放区;**不建真指针** → 天然无不纯指针、无幻象碎片、换出不留二级锚(行为=假指针的效果)。
- **一个存放区**:assemble_context 把 retrieve() 的所有 hits(RAG/L2)`page_in` 进同一个 `ContextWindow`。

❌ 差的(= 本次两个任务要解决的):
- **"缓存"一词重载**:存放区(队列占用) / `NeighborCache`(索引区 L2 翻图加速) / LLM prompt KV 缓存,三个不同东西都被叫过"缓存",曾让用户误以为"缓存空了=记忆坏了"。
- **没有显式的"假指针"对象、没有 LLM/RAG 指针标签** —— RAG 命中是隐式地"不建指针",而非显式地"建假指针统一进队列"。
- 面板曾把 `NeighborCache` 当"记忆缓存"摆出来(本会话已临时撤,见 §5),但没系统化。

## 3. 任务①:记忆系统 — 统一概念(代码)
> 目标:让代码结构/命名**显式反映 §1 的统一模型**,消除"缓存"重载。先对设计(本指南+架构文件)再动,改的是记忆内核要谨慎。

- **3.1 术语统一(贯穿 core/memory/gui/前端注释与命名)**
  - 存放区/临时记忆 → 统一概念**「工作区 / 队列占用」**(working region),代码 = `ContextWindow`。
  - 索引区 → **一级指针(磁盘 `store.rs` EDGES)+ 二级锚(`secondary.rs`)+ RAG 向量索引(`index.rs`)+ 索引加速缓存(`cache.rs` 原 `NeighborCache`)**。
  - `NeighborCache` → 注释/(可选)改名明确为**「索引区·L2 翻图加速缓存」**,绝不叫"记忆缓存"。(i18n 本会话已改显示名「L2导航缓存」。)
- **3.2(建议,核心)显式化 RAG"假指针"**
  - 现状:RAG 命中只 `page_in`、无指针对象。
  - 改:给 RAG 命中在**指针队列**里建一个带 `kind=Rag/fake` 标记的**假指针**(`pointer.rs` 加 kind 或 flag);**约束铁律**:假指针换出队列时**不落序列(不建二级锚)、不进碎片系统**。
  - 价值:把"RAG 命中也在缓存队列里"从隐式(page_in)变显式(可统一置换/可被面板计数),严格贴合用户原话"将 RAG 的指针作为假指针方便统一在队列和缓存中"。
  - 落点:`pointer.rs`(指针 kind)、`retrieval.rs` retrieve() 的 RAG 命中分支、`memory/mod.rs`(计数/暴露给面板)。
- **3.3(可选,随 3.2)LLM/RAG 指针标签**:真指针标 LLM(走碎片回收)、假指针标 RAG(不回收)。多数行为已天然如此,这步是把它**显式化**便于面板区分与未来 §1(B) 的"RAG 暴力扫则给它单独染色"扩展。
- ★决策点:3.2/3.3 改记忆内核,需用户确认是否做(还是仅做 3.1 命名统一)。
