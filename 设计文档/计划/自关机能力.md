# 自关机能力 — 设计稿(2026-06-04,待实现)

> 用户原话见 `用户决策/决策日志.md` 2026-06-04 条。本文给设计,实现留续点(0-OPUS22 欠项①)。
> 本质:GrowBox 能"把自己关掉"+ 能触发系统关机,经一次性临时授权(或设置里的永久权)全自动执行。
> 归入既有安全门体系(`设计/03-安全审查`),不另起炉灶 —— 关机是一类高风险动作,套用 shell 授权的"拥有者裁决"原则。

## 1. 两种"关闭"(分清楚)

- **关闭自己(进程退出)**:GrowBox 进程结束。Tauri 提供 `app.exit(code)`。这是"我下班了"。
- **系统关机**:关掉整台机器。走 shell `shutdown`(mac:`shutdown -h +N` / 立即 `halt`)。这是"机器也关"。
- 用户场景:做完任务 → 安排好系统关机(脚本)→ 关闭自己。两步,顺序关键(见 §4)。

## 2. 授权模型(对齐 shell_gate 的"拥有者裁决")

凡"含关机命令"的动作(关闭自己 / 执行 shutdown 类 shell)默认**弹一次性临时授权**:
- **一次性临时授权**:弹窗批准 → **只生效一次**(本次动作);下次再关仍弹。登记表记"本次已授权",用完即焚(不进持久信任表)。
- **永久权(设置)**:`Settings.auto_shutdown_allowed`(默认 false)。为 true 时关机动作免弹窗、全自动。
- 没请求过的关机动作 = 一律弹窗(安全侧)。复用 `approval.rs` 登记表 + 一个新裁决类型(区别于 shell 的 AllowOnce/AlwaysThis/TrustAll —— 关机的"AlwaysThis"就是去设置开 auto_shutdown_allowed,不在弹窗里给持久选项,避免误点永久放权)。

## 3. 执行器(架构公理:一切能力皆执行器)

- 新执行器 `shutdown`(或 `power`,参数化):`{action: "exit_self" | "system_shutdown", delay_secs?}`。
  - `exit_self`:`app.exit(0)`。
  - `system_shutdown`:走 shell `shutdown`(经 shell_gate + 关机授权双门)。
- `Risk` = 新增最高档(关机不可逆,必裁决);`shutdown_gate`(类比 shell_gate)永远先于一切,硬底线。
- emit 一个 `PermissionRequest{kind:"shutdown"}` 让前端弹一次性授权窗(前端按 kind 分流,红框警示)。

## 4. 提示词引导(★关键,用户特别强调★)

系统提示词(system.{zh,en}.md)须写明:
- **关闭自己之前,先把后续安排好**。因为 **Mac 关机时控制台/登录会话可能阻止关机**(进程没退干净、确认弹窗),所以正确姿势是:
  1. 先 **detach 一个倒计时关机脚本**(如 `shutdown -h +1` 或 `nohup` 一个延时脚本),让它独立于 GrowBox 进程存活;
  2. 确认脚本已启动(probe);
  3. **再关闭自己**(exit_self)。
- 即"安排好身后事再走":不要直接 `shutdown -h now` 然后自己也被杀导致脚本没起来;也不要只关自己不关机(若任务要求关机)。

## 5. 落点清单(实现时)

- `core`:`Settings.auto_shutdown_allowed`(默认 false)+ 风险档。
- `safety`:`shutdown_gate`(关机命令识别 + 授权裁决);关机命令黑/识别名单(shutdown/halt/poweroff/`app.exit`)。
- `gui`:`executors/shutdown.rs`(exit_self 调 app handle;system_shutdown 走 shell_gate+shutdown_gate);`PermissionRequest.kind="shutdown"`;一次性授权登记表。
- 前端:`PermissionDialog` 加 shutdown kind(红框 + 二次确认 + "本次授权"措辞,不给"永久"按钮,引导去设置);Settings 加「允许自动关机」开关。
- 提示词:system.{zh,en}.md 加 §4 引导。
- 测试:授权门(无权弹/有权过/一次性用完即焚)+ 执行器分发。

## 6. 安全边界(诚实)

- 关机是**不可逆 + 影响整机**的最高风险,默认必须人确认(一次性),只有用户在设置里显式开永久权才全自动 —— 决定权留人(对齐 `设计/03` 原则1 拥有者裁决)。
- 永久权开启后,GrowBox 可在"做完保命"后自主关机(用户最初场景)。但这是用户主动授予的信任,非默认。
