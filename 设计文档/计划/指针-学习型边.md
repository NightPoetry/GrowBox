> 关联:`计划/precision-layer.md` 阶段1(指针网原型)、`计划/storage-standards.md`(EDGES 磁盘布局)、做梦/碎片维护层(`设计/02` 维护节)。

## 目标
把一条指针边从"单 K 覆盖、无负样本"升级为**带正负 K 的学习型边**:边越被成功复用越"万能",误跳被记成反样本下次自动规避;匹配提供"廉价加权余弦"与"LLM 综合判断"两档,按推论9 可调。

## 现状(起点,代码核过)
- `pointer.rs`:`Pointer { target: String, topic: Vec<f32>, heat: u32 }`。`link(source,target,topic)` 对已存的 (source,target) 边 **`heat+=1` 且 `topic=本次Q`(覆盖)**——历史 Q 全丢。
- `memory.rs::retrieve_exact`:`embed(Q)` → 指针网入口取 top-K 节点 → 取该 source 邻域(`NeighborCache`,平铺单 LFU)→ 每条出边 `cosine(Qv, topic) ≥ 指针跟随阈值` → **judge 目标 1 次确认** → 命中 bump+染 Light+短路;judge 拒 → 继续 / 线性兜底。
- **关键缺口**:judge 判"不相关"的信息**被丢弃**了(本设计把它捡起来当反 K);单 topic 丢历史(本设计累积成正 K 组)。

## 原理(为什么)
- **多正 K**:一个 target 通常对应一**族**提问;累积历次成功的 Q(各带权重)= 在线学这条边的"查询簇",比单个最新 Q 更稳、召回更全。
- **反 K = 硬负样本挖掘**:误跳过一次就记住,下次相似 Q 直接规避——省 judge、防误召回。这是现有机制完全没有的学习信号。
- **两档匹配**:廉价向量档覆盖大多数;拿不准时升 LLM 综合判断档(判"边"而非逐 target,且能读负样本)。呼应 `设计/02` 原则1"成本最优级联"。

## 数据结构(Pointer 演进)
```rust
struct KSample {
    text: String,    // 当时的 Q 原文(LLM 综合判断 + 可观测;旧边迁移时可空)
    vec: Vec<f32>,   // Q 向量(加权余弦档用)
    weight: u32,     // 命中/复用次数(越多越响)
    last_used_ms: i64, // 冷热(做梦合并/淘汰用)
}
struct Pointer {
    target: String,
    positives: Vec<KSample>, // 正 K:历次成功命中的 Q(替代旧单 topic)
    negatives: Vec<KSample>, // 反 K:judge 判不相关的 Q(硬负样本)
    heat: u32,               // 边总热度(≈ Σ positives.weight);平铺缓存仍按它淘汰
}
```
**迁移**:旧 `topic: Vec<f32>` → `positives = [KSample{ text:"", vec:topic, weight:heat, last_used_ms:now }]`。旧边无 text → 自动降级只能走加权余弦档,新命中起补 text。EDGES 的 value 序列化结构升级 → **加 schema 版本号**,读旧版按上式迁移(见 `storage-standards`)。

## 匹配算法(当前 Q 要不要跟这条边跳到 target)
**档 A — 加权余弦(默认,廉价)**
- `pos_score = max over positives k: cosine(Qv, k.vec) × (1 + ln(1 + k.weight))` —— 取最响的正 K(防一个高频 K 单独把边变磁铁:用 max 而非求和,权重只放大不叠加)。
- `neg_hit = ∃ negative n: cosine(Qv, n.vec) ≥ NEG_BLOCK_THRESHOLD` —— **反 K 一票否决**(保守:宁漏不误召回)。[可调:阈值;或改"最近邻 K 定符号"]
- **跟随** iff `归一(pos_score) ≥ POINTER_FOLLOW_THRESHOLD 且 !neg_hit`。
- 命中后是否仍 judge 1 次确认 = `FORCE_JUDGE_ON_COSINE_HIT` [可调:精确 vs 省钱]。

**档 B — LLM 综合判断(精确)**
- 把这条边的 positives(原文+权重)、negatives(原文)、当前 Q 原文一起喂潜意识 judge:
  "这条边曾被这些提问命中[正K列表(权重)]、被这些拒绝[反K列表];当前提问=Q。是否值得跳到 target?"
- 一次 LLM 调用产出 跳/不跳 —— **判"边"而非逐 target judge**,且利用负样本。比现有"逐 target judge"信息更全。

## 学习更新(每次检索的副产品,接 `设计/02` 原则2 飞轮)
- **命中**(judge 确认相关 / 档A 采纳):当前 Q 入 positives。去重:与最近正 K `cosine ≥ K_MERGE_THRESHOLD` 则该 K `weight+1`、刷新 last_used(**保留那个真实 K,不合成簇心**);否则 push 新 K。这一步就是"有界"的第一道闸(见下)。
- **judge 拒**(Q 走到这条边但 target 不相关):当前 Q 入 negatives(同样去重)。→ 下次相似 Q 被反 K 挡。

## 可调旋钮(推论9,进控制面板/设置)
- `pointer_match_mode`: `weighted_cosine` | `llm_judge`(档A/档B)。
- `pointer_follow_threshold` / `neg_block_threshold` / `k_merge_threshold`(数值)。
- `force_judge_on_cosine_hit`(bool)。
- 默认:weighted_cosine + force_judge=true(向量预筛 + judge 确认,≈现状行为但带负样本与多正 K)。

## 有界(纯统计:全是真实过去 Q,无合成、无 LLM)
> 设计铁律(用户 2026-06-02):防 K 膨胀必须**保持原设计的纯粹——K 永远是真实出现过的 Q、零主观判断、全是过去统计**。故**不做**"做梦合并成簇心"(簇心=加权平均=从没真实出现过的虚构 Q,合成物)与"LLM 找父节点描述"(主观)——两者均毙。在线纯统计两道闸即可:

1. **近似坍缩(去重,保留真实 K)**:新 Q 与某已存正 K `cosine ≥ K_MERGE_THRESHOLD` → **不新增,给那个真实 K `weight+1` + 刷新 last_used**。near-identical 提问坍缩到**一个真实过去 Q** 上,weight=代表次数;只有真正不同的查询模式才各占一个 K。(反 K 同理。)
2. **LFU + 近因封顶**:仍超 `cap N`(默认 8)[可调] → **淘汰最冷的真实 K**(weight 最低 + last_used 最旧),同邻域缓存淘汰最冷。丢的是"很少再被问到的旧模式",纯频率/近因统计;留下的每个 K 仍是真实 Q。

→ 全程无平均、无合成向量、无 LLM;膨胀被坍缩+LFU 压住。**做梦对 K-bounding 不再必要**(做梦回归本职:扫碎片补漏)。
- **可选的"代表/父"= medoid**:若将来要一个代表这簇 K 的节点,用 **medoid**(对其他 K 余弦总和最大的**那个真实 K**=最居中的真实过去 Q),由纯统计选出,不是合成簇心、更不是 LLM 描述。默认**不做分层**(坍缩+LFU 已够),medoid 留作"要显式代表"时的纯统计选项。
- 推演阶段(Sleep)可**复核反 K**:被反 K 长期挡住的边偶尔放行 judge 一次,纠正历史误判(防一次误判永久封路)。此为"维护"非"bounding",仍只增删真实 K。

## 边界(诚实记录)
- 旧边无 text → 档B 对其降级为档A,直到新命中补上 text。
- 存原文增存储:K 文本=Q(短)、cap N 有界,可接受;落 redb EDGES value(结构升级需版本+迁移)。
- 反 K 一票否决可能漏召回(被历史误判挡):靠"阈值可调 + 做梦复核反 K"兜底。
- max-sim(而非求和)是防过度泛化的有意选择;若实测召回不足,再调成 top-k 加权和。

## 配套:检索动作记成时间线一等事件(自我感知原则第三次泛化)
> 用户 2026-06-02:"用户操作 UI 都能在时间线中有地位,为啥 search 不能有?" + "这也不算多检测一个节点,毕竟它跟普通消息就是标签不同"。
- **走现有 `perceive` 同一通道,不另起机制**——同 UI 面板开关(`note_ui_panel`→perceive)、后台任务(#4)。检索事件 kind = **`mind_search`**(用户命名);`retrieve`/`retrieve_exact` 后 `perceive("mind_search", "下沉检索「Q」找到 N 条")` → 落时间线节点 + LLM 下一回合可感知"我刚检索了什么、找到了谁"。
- **节点类型 = 受控取值表(用户拍板,收敛点)**:`Node.role` 本是自由字符串(加 `mind_search` 即可用);收敛为**一张受限 kind 表**(user/assistant/system/internal/`mind_search`/ui_event/task_event…),节点**只存索引/code,仅加载进上下文时按表恢复**成 LLM 能懂的标签。= `notices.i18n.json` catalog 模式(code→中心渲染、用时展)推广到时间线节点 kind。**诚实**:就单 tag 内联省存储而言,相对每节点 content+384维 embedding(~1.5KB+)可忽略;真价值=受控词表(防散装)+ 含义单一源 + 用时恢复。是个**地基微调**(role 字段语义收敛为受控 kind,旧值天然合法,无破坏性迁移)。
- **不在记录时过滤**:检索节点跟普通消息只差标签,一律记(标签携带层+结果);噪音是消费时按标签筛的事(同 role=internal 单独渲、不混正文),不是少记。只记摘要不记全遍历。
- **与学习型指针的闭环**:正/负 K = 这些真实检索事件的索引;K 淘汰零损失,因 Q+检索+结果都在时间线躺着(truth),指针(K)只是其上的可再生加速层。
- **实现**:本身就是 perceive/notice 加一个 code + 一处调用,很小;可与学习型指针同批,或先单独落(更快见效,且为指针的"K↔真实检索事件"提供干净审计)。
- 自我感知三泛化:**UI 操作 / 后台任务衍生物 / 自身检索**,AI 对自身及衍生的一切行为绝对可感知(接 `internal-state-perception` / `感知告知-双受众`)。

### 阶段3:档A 加权余弦 + 反 K 一票否决(2026-06-02 Opus)
- `pointer.rs`:`follow_score(qv) = max over 正K: cosine × (1 + GAIN·ln(weight))`;`neg_veto(qv, thr) = 任一反K cosine ≥ thr`。
- **偏差(工程化,如实记)**:规格字面 `factor=1+ln(1+weight)` 对 weight-1(刚建未复用的边)也放大 factor≈1.69,会把跟随门从 cosine 0.80 普降到 ~0.47、过度放宽 + 暴涨 judge。改为 `1 + GAIN·ln(weight)`(`POINTER_WEIGHT_GAIN=0.3`[可调]):weight-1 → ln(1)=0 → factor=1 = **完全保留基线 cosine 门**,只让真正被复用的热 lane 变黏。`max`(非求和)防单个高频 K 把边变磁铁(知识保留)。
- `memory.rs`:retrieve_exact 跟随门换成「先 `neg_veto`(`NEG_BLOCK_THRESHOLD=0.90`[可调],近似重复才否决、省 judge)→ 再 `follow_score ≥ POINTER_FOLLOW_THRESHOLD」`,替换旧 `cosine(qv, primary_topic)` 单门;移除 memory.rs 不再直接用的 `cosine` 导入。`primary_topic` 保留(pub 访问器,未来 UI/调试)。
- 测试:memory **75** 绿(pointer +3:weight1=裸cosine / max+热lane加权 / neg_veto 只挡近似;集成 +1 `neg_veto_blocks_mesh_recall_of_deep_target` 带对照组隔离否决因果——用私有 `record_negative_edge` 注入反K);clippy 零;全工作区绿。
- **注**:judge 仍是真精度门;档A 只是更廉价/更高召回的预筛(读查询簇 + 热 lane 变黏),反 K 否决在 judge 前剪掉已知误跳、省 judge。被反 K 长期误挡的纠正留阶段6 sleep 复核。

### 阶段4:档B LLM 综合判断(2026-06-02 Opus)
- `subconscious.rs`:trait 加默认方法 `judge_edge(query, positives:&[String], negatives:&[String], target) -> bool`。**默认回退**到对 target 单条做 `judge_relevant`(= 逐 target 行为,所有 mock/无 LLM 自动得合理默认,无需逐个改实现);签名用 `&[String]`(memory 层把 KSample 格式化成"原文(命中N次)")而非 KSample,使 trait 不依赖 pointer 内部类型。
- `bridge.rs`:`LlmBridge` override `judge_edge`——prompt 列出正 K(原文+命中次数)、反 K(原文)、target 内容,要 LLM 输出 `{"jump":bool}`;调不通/解析失败=保守不跳(精确档宁漏)。
