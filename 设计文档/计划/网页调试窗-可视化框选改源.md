> 状态:2026-06-10 起做,git feature/artifact。**Phase 1 + Phase 2 已落地、真机端到端调通(commit cd9acd2 + 27b6271)**;反向定位精度=下一步(归 Skill,见 [[skill-system-direction]])。
> 记忆:[[web-debug-select-to-edit]] + [[skill-system-direction]]。复用造物体系(见 [[被造物-artifact-implemented]])。

## 本质

对"正在开发的网页东西"做**可视化框选改源**:横栏点套索 → 在渲染面里框选/点选一个区域 →
hover 高亮、收 DOM 快照(选择器路径 / outerHTML / bbox)→ 选框旁输入修改建议 →
后端据 DOM 快照 + 相关源码定位改哪 → 改源 → 重渲。闭环。

**关键:目标里的"URL"= 一个真在跑的本地 web 应用入口**(如自己写的 Node 站跑 localhost:3000),
共同点 = **源码本地、可真改**;任意外网站点(无可编辑源码)排除。

## 架构:两渲染面、一套共享调试运行时

套索绘制 + 命中 DOM + 快照序列化的 JS **只写一份**,两处注入:

| 目标 | 渲染面 | 注入方式 | DOM→源码 |
|---|---|---|---|
| 造物 | 现有沙箱 iframe(`ArtifactCanvas.tsx`,srcdoc + GX_BOOTSTRAP) | srcdoc 前缀 | AI 自己写的 HTML,直定位(在对话/.growbox/artifacts/) |
| 本地 URL | **新建真 Tauri WebviewWindow**(沙箱 iframe 加载不了 URL、跨源注入不进) | Tauri `initialization_script` 页面层注入;**回传走 `gxlasso://` 导航哨兵 + `on_navigation` 拦截**(★Tauri 对 External URL 的 IPC 不可靠、且被调试页 CSP `default-src 'self'` 拦跨源 fetch=Load failed;CSP 的 connect-src 管不着导航,故用导航回传★) | 编译产物→源码,AI 靠 class/文本/结构用 code_search 反查(非 1:1,AI 推断)→**精度升级=注入 dev-inspector 插件带 `data-source` 坐标,归 Skill** |

## 分相

### Phase 1 — 造物上的框选改源(全在现有 iframe,不碰新 webview)★已落地(待真机目测)
> 实现全在单文件 `ArtifactCanvas.tsx`(硬编码中文跟该文件既有惯例;零后端改动)。横栏「⌖ 套索」按钮 →
> GX_BOOTSTRAP 内 gxStart/gxStop 调试运行时(overlay+hover 高亮+点选/拖框选+gxSelector 选择器路径+gxSnap 快照)→
> postMessage `inspect-select` → onMsg setInspectSel → 底部"修改建议"输入面板 → submitVisualEdit 拼消息走 doSend。
- GX_BOOTSTRAP 加调试运行时:收 `{__gxcmd:'inspect',enable}` 进/出选择态;
  hover 高亮元素;点选单个 / 拖框选多个;收快照 `{rect, elements:[{selector,tag,id,classes,text,outerHTML(截断),rect}]}`;
  postMessage 回 `{__gx:true, type:'inspect-select', selection}`。
- 前端:`artifactInspect` 信号 + `setArtifactInspectMode(enable)`(postMessage 进 iframe);
  横栏加"套索"切换按钮;`onMsg` 收 `inspect-select` → 弹选框旁的"修改建议"输入框(可信 SolidJS 层,仿 notice)。
- 提交:把 {建议 + DOM 快照 + 指令} 拼成一条消息走 `doSend` → 普通 Agent 循环 → AI 读自己渲染过的造物 HTML、
  定位、`render_artifact` 重渲。**零后端改动**(MVP)。
- i18n 四语齐(按钮/占位/发送)。
- 完整可演示:造物 → 套索点按钮 → "改绿" → AI 重渲。

### Phase 2 — URL 面(真 Tauri webview)★已落地真机调通(commit 27b6271)
> 用户点透本质=「调试网站」大白话 → AI 自动起 dev server → 调试窗加载 URL → 套索框选 → 改**本地源码** → 刷新。
- **执行器/命令**:`open_debug_url` 执行器(ui_intent 往返)+ `create_debug_webview`/`reload_debug_webview` 命令(`web_debug.rs`);真 `WebviewWindowBuilder` + `initialization_script` 注入 `web_debug_runtime.js`(注入版套索运行时,自带 shadow-DOM 工具栏「⌖套索」「⟳刷新」+ 输入框);`tools.i18n.json` 的 open_debug_url llm_desc 含「调试网站」编排指引(AI 读 package.json→shell 起 dev server→抓 URL→开窗)。前端 `ui-actions.ts` dispatch + `App.tsx` listen `web-debug-edit` → `chat.ts sendWebDebugEdit` 走 doSend。
- **★回传绕 CSP(关键)★**:被调试页常带 CSP `default-src 'self'`,跨源 fetch 被拦(实测 `Load failed`);Tauri 对 External URL 的 IPC 也不可靠。→ 注入脚本把框选 JSON 塞进 `gxlasso://x/?d=...` 触发导航,`on_navigation` 拦下取数据 emit、返回 false 取消导航(页面不动)。**connect-src 管不着导航 → 任何带 CSP 的页面都通**。
- **无 HMR(EJS/Express)**:AI 改完源回合结束 → 自动 `reload_debug_webview`(eval location.reload,Tauri 原生注入不受 CSP 限);+ 工具栏「⟳刷新」手动兜底。
- **诊断脚手架(官方包待去)**:Info.plist 加 `NSAllowsLocalNetworking`(ATS,非元凶);Cargo `devtools` feature + 调试窗 `open_devtools()`;运行时 HUD(点数/选中/状态)+ 选中绿高亮。
- **真机验证**:博客 EJS,套索页脚 → "My Blog"→"NightPoetry" 跨 4 文件全改、刷新见效。
- **★反向定位精度=下一步、做成第一个 Skill★**:现在 DOM→源码是 code_search 猜(静态能中,动态 `<%= %>` 不准)。正解=注入 dev-inspector 插件(react-dev-inspector/vite-plugin-vue-inspector/vite-plugin-dev-inspector,渲染元素带 `data-source` 坐标,Trae/v0/Lovable 的做法)。这是**知识不是机制**→归 Skill 系统(见 [[skill-system-direction]]),别硬编码成框架专用执行器。
