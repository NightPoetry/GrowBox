# 自驱续跑(Self-drive · 全自动模式下的"一直跑")

> 落地:2026-06-14(0-OPUS39),git `feature/artifact`。本文 = 设计 + as-built 合一。
> 一句话:全自动模式下界面解锁一个循环按钮,点亮后**每当 AI 自己停下来,系统就自动注入一条"继续推进"种子**让它评估现状、判断要不要重构/治理屎山、动手做下一步,如此一直跑下去;直到用户喊停或确实没事可做。

---

## 1. 需求(用户原话,2026-06-14)

> "我们不是有全自动模式吗?全自动模式下可以在界面上解锁一个按钮,是一个循环 svg 图标,在发送框的左边……点击这个按钮就会被激活,激活后每当程序自己停下来了,就会自动询问接下来做什么,是否需要重构,如何避免变成屎山代码,并且采纳建议,然后自己继续做,让程序可以一直跑一直跑。"
>
> "当然这种自动鞭策肯定是不写进历史记录中的,但是在记录中。即除了真正的用户的交互内容和 AI 回复,只进记录不进历史记录(实际上就是个标签问题,你看看项目自然就懂了)。"

拆成三条硬要求:
1. 全自动模式下界面解锁一个循环按钮(发送框左侧),点击 = 激活/取消(开关)。
2. 激活后每当 AI 停下来,自动鞭策它继续(评估现状 / 要不要重构 / 怎么避免屎山 / 采纳建议 / 接着干),一直跑。
3. **自动鞭策的提示词只进"记录"、不进"历史记录"** —— 用户点明这是个"标签问题"。

---

## 2. 核心机制:标签 = `role`(用户说的"标签问题")

每条时间线节点(`growbox-memory` `Node`)带一个 `role` 字段。两处消费按 role 分流:
- **对话历史**(`cmds/history.rs` `get_chat_history`):**只放 `role == user | assistant`**,其余一律过滤。
- **记录 / 时间线 + AI 感知**(`assemble_context` 的 recent ring / RAG):**所有 role 都进**。

于是"只进记录不进历史"= 把鞭策种子以 `role = internal` 落库:
- 鞭策种子 → 内部消息路径 → `ingest_with_role(seed, "internal")` → **进记录、AI 全链路可感知、可检索**,但被 `get_chat_history` 的 role 过滤挡掉 → **不进对话历史**。✅ 第 3 条。
- AI 自己的回复/动手 → 仍以 `role = assistant` 落库(脊柱 `agent/mod.rs` 收尾处 `ingest_with_role(final_text, "assistant")`)→ **进历史,用户照常看得见它在干活**。✅ "真正的 AI 回复进历史"。

这正是用户说的"看看项目自然就懂了":机制早已存在(内部消息 vs 用户消息,见 [[内部状态感知]] / 决策日志 2026-06-02),自驱只是复用它。

另:受控 kind 表(`node_kind.rs`)新增 `SELF_DRIVE` kind(非散装自由串),给记录里这条内部种子一个清晰双语标签("自驱续跑 / self-drive"),`role` 仍是 `internal`。

---

## 3. 架构:前端编排 + 后端单步命令(**故意不做成后端 worker**)

每个自驱轮 = 一次独立的后端命令调用 `self_drive_step`(各自取仲裁档 + AppState 锁、跑完即释放),由**前端循环器**(`chat.ts`)串起来。

**为何前端编排(用户 2026-06-14 拍板"保持就好")**:
- **资源每轮间能喘息**:每轮 `run_chat` 独立取/放锁与仲裁档 → 轮与轮之间 idle 学习 / 做梦整理跑得起来(这本身就是"避免屎山"的一环)。后端大循环会把仲裁 Agent 档攥住不放,idle 飞轮永远轮不上。
- **UI 实时可见 + 可插话**:每轮自然流式渲染;用户随时能按「终止」或在续跑间隙插入新指令。
- 代价:核心编排落在被保留的"前端交互层"里,没那么"脊柱纯粹"。但**没有另开驱动路径**——每轮仍经 `run_chat → agent_loop_internal` 唯一脊柱(类比常驻 Supervisor 的"外部决定何时再调脊柱")。

> 备选(未采用):做成后端 `SelfDriveWorker`(类比 `supervisor.rs`)更"脊柱一致",但失去上面三个好处。如将来要改,这是落点。

### 数据流(一个自驱轮)

```
前端 runSelfDriveLoop 循环体
  └─ api.self_drive_step()                         [cmds/chat.rs]
       ├─ 仅 auto_mode=true 才驱动(否则空返回让循环停)
       ├─ self_drive_seed(lang)  → 中/英鞭策种子(后端自建,前端永远看不到=绝不进历史)
       ├─ CountingSink 包 TauriSink(数"真干活"工具调用,排除 finish/ask_user)
       └─ run_chat(seed, internal=true, self_drive=true)
            └─ perceive(SELF_DRIVE, seed) + agent_loop_internal
                 ├─ 种子以 role=internal 入库(记录,不进历史)
                 └─ AI 回复/工具以 role=assistant 入库(历史可见)+ flywheel finalize(学习)
  └─ 返回 { content, did_work }
       ├─ did_work=true  → 继续下一轮
       └─ did_work=false → 连续 N 轮(默认2)判"没事可做"→ 自动暂停 + toast
```

### 停止三途
1. 用户按「终止」(`cancelSend` → 关 `selfDriveActive` + `cancel_chat`):当前轮脊柱检查点优雅收口,循环停。
2. 关全自动模式 / 断连(`App.tsx` effect 强制复位 `selfDriveActive`):按钮随 autoMode 隐藏,循环停。
3. 连续 N 轮 `did_work=false`(AI 只回话不动手=确实没事可做):自动暂停 + toast,再点循环按钮可续。

**退出信号设计**:鞭策种子明确告诉 AI"确实没事可做就**不调任何工具**只用一两句话说明现状"→ 无工具调用 = `did_work=false` → 触发暂停。既给 AI 干净的退出口,又不靠脆弱的文本哨兵。

### 用户插话优先
续跑期间用户在发送框敲消息(`doSend` 见 `sending` 把它排进 `msgQueue`)→ 循环体每轮先查 `msgQueue`,非空则**停自驱、先把用户的话当正常回合处理完**(用户主动指挥永远盖过自动续跑)。

## 4. 可调旋钮(用户铁律:一切涉及数值的都应能设置)

纯前端循环参数,`localStorage` 持久、即时生效(`store.ts`),每轮现读 → 改了立刻生效、越界回退默认。UI 在 **设置 → 智能体 → 「自驱续跑」区**(两个 number input,四语)。

| 旋钮 | 默认 | 含义 |
|---|---|---|
| `selfDriveIdleLimit`(空转暂停轮数) | 2(最小 1) | 连续多少轮 AI 没动手就判没事可做、自动暂停 |
| `selfDriveGapSecs`(每轮间隔秒) | 1.5(最小 0) | 每轮之间喘息,给 UI 刷新 + 后台学习/整理缝隙;0=不间断 |

> 间隔旋钮也是对"防账号过热"的一个用户可控阀(见 [[phase-pacing-cooldown]] 的同源关切,但那是 dev 工作流规矩,这是运行时阀)。

## 5. 设计哲学符合性自查(用户 2026-06-14 要求按其哲学)

- **提示词多语种**:鞭策种子按 `prompt_lang` 出中/英两套(比现有 artifact/terminal 内部种子只有中文更到位)。见 [[i18n-two-axis]]。
- **界面文案四国**:`selfDrive*` 九个 key 在 zh-CN/en/ja/zh-TW 全 parity。
- **禁止 Emoji**:提示词/文案零 emoji,按钮是 SVG(repeat 图标)。
- **一切数值可设**:`idleLimit`/`gapSecs` 从写死常量改成可调旋钮(见 §4;[[numeric-settable-and-cache-flat]])。
- **单一源文案**:tooltip/toast 全走 `t()`。
- **能触发就触发不轮询**:循环事件驱动(每轮由上一轮完成触发),间隔只是喘息不是轮询。
- **受控 kind 表**:新增 `SELF_DRIVE`,非散装。
- **不绕脊柱**:经 `run_chat → agent_loop_internal` 唯一脊柱,`CountingSink` 包 `TauriSink`(全方法透传)。

## 8. 健壮性加固(2026-06-14 续,P1-P5;基于一轮自查发现"× N 轮"维度放大的隐患)

第一版每条单轮逻辑都对,但"持续循环"这个新维度把原本无害的东西放大了。五条加固:

- **P1 跨轮进度指纹 + 总轮数软上限**:第一版的暂停只认"没调工具"的轮;但 AI 陷入"每轮调个没用工具→finish"的退化循环时 `did_work` 恒真→永不停→整夜烧 token。加固:`chat.ts` 每轮把流式全文规范化成指纹(`fingerprintRound`),"无进展"判定扩成 **没动手 或 与上轮指纹相同**;连续 `idleLimit` 轮无进展→暂停(toast 区分"没事做"vs"原地打转")。+ `selfDriveMaxRounds` 旋钮(默认 0=无限,到顶暂停交还用户)。
- **P2 周期性消化**:持续自驱时每轮 `run_chat` 两次 `touch_activity` → idle 计时器永不到 8 分钟 → flywheel 压缩/做梦永不触发 → 经验只采集不压缩堆积(讽刺:这功能本意之一是"避免屎山/治理债",却把自己的学习消化饿死)。加固:后端 `run_digest_pass` 命令(一次性飞轮消化,复用 `idle::digest_while_idle` 同组 Flywheel 原语,去掉 idle 让位/skill 提议);`chat.ts` 每 `selfDriveDigestEvery` 轮(默认 12)主动调一次。
- **P3 鞭策种子不落 timeline**:见 §2 末——每轮只记一条精简 SELF_DRIVE 标记进记录,完整 ~500 字种子只作本轮系统消息;防几百轮重复种子膨胀 timeline + 污染 recent-ring。(`run_agent_loop` 加 `self_drive` 参数跳过全文 ingest;`run_chat` perceive 精简标记。)
- **P4 danger 模式(为所欲为)**:用户决策——全自动只能在项目内做事,系统级需求(系统装 Python 等)会卡授权弹窗、无人值守时超时转拒。新增最高放行档:`Sandbox.danger` 标志使 `judge` 一律 Allow(绕过越界/敏感路径/危险命令/SSRF);`shell_gate` danger 早返回(跳过 LLM 审核+隐私网)。`AgentConfig.danger_mode` + `Settings.danger_mode`(**`serde(skip)` 不持久**,每次启动默认关,防遗忘的危险模式自启);`run_chat`/`supervisor` 每回合从 settings 同步 `cfg.danger_mode` + `sandbox.set_danger`;`set_danger_mode` 命令即时同步 sandbox。前端 danger 开关(**会话级不持久**,强红警告,仅全自动开启时显示;关全自动/断连→App.tsx 强制收掉,防"开关随 auto 隐藏但后端仍开着"的陷阱)。**安全权衡**:这是用户明知故犯的最高档,文案三处强调极高风险 + 不持久,把"忘记关"的窗口压到最小;但开启期间确实无任何拦截。**★全局红边视觉信号(用户 2026-06-14 加)★**:danger 开启时 `App.tsx` 给 `<body>` 挂 `.danger-active`,`index.css` 把全局线条/边框变量(`--separator`/`--border`/`--border-color`/`--border-light`,被 69+ 处规则引用)染微红 + `body::after` 整窗红框内侧微光——所有线条一起变,持续提示"危险中"。变量级覆盖、零侵入。
- **P5 默认间隔调大**:`selfDriveGapSecs` 默认 1.5→3,降 API 速率(关切见 [[phase-pacing-cooldown]]);总轮数软上限复用 P1 的 `selfDriveMaxRounds`。

新增旋钮(全在设置→智能体「自驱续跑」区,四语):`selfDriveMaxRounds`(默认0)、`selfDriveDigestEvery`(默认12);gap 默认改 3。danger 开关在设置→安全(全自动区下)。

新增文件改动:safety `sandbox.rs`(danger 标志+judge短路+set_danger+测试)、`shell_gate.rs`(danger早返回)、`agent/types.rs`(danger_mode)、`agent/mod.rs`(self_drive 参数)、core `project.rs`(Settings.danger_mode serde-skip)、`cmds/status.rs`(set_danger_mode)、`cmds/history.rs`(run_digest_pass)、`supervisor.rs`、`main.rs`(注册 2 命令);前端 `store/tauri-api/chat/AgentTab/Settings/App` + i18n 四语 + css danger 样式。
