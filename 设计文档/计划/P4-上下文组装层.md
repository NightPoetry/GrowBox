# P4 上下文组装层(记忆置换系统的"换入上下文"那一环)

> 真理来源:`设计文档/记忆置换系统-总纲.md` + `用户决策/决策日志.md`(用户原话逐字)。
> 本层消费 P3 精确层(指针网/三级缓存/时间线/向量索引),把无界记忆映射进有限 LLM 上下文窗口。
> 注:此处 P4 是项目"路线"编号(P0嵌入/P1异常告知/P3精确层磁盘原生/P4组装层),
> 与 `precision-layer.md` 的"阶段1-5"(精确层内部:指针/缓存/染色/碎片/做梦)是两套编号,别混。

## 落点(架构最优,2026-06-01 定)

- **置换/页表策略放 `growbox-memory/src/context.rs`**:常驻工作集、两态、ring、按指针调入、预算淘汰。
  llm 无关,产出抽象 `ContextBlock`,可在 memory crate 内单测。
- **gui 侧只做提示词工程**:把 `ContextBlock` 套上"每区独特标记 + 区内角色说明 + 时间戳 +
  按时间戳判序"的外壳,转成 `ChatMessage`,按稳定→易变排序。
- 理由:memory 不依赖 llm(保持分层);置换是记忆系统本体,formatting 是 gui 提示词关注点。

## 上下文分区(稳定→易变,为 prompt 缓存)

| 序 | 区 | 谁持有 | 缓存性质 |
|---|---|---|---|
| 1 | system prompt(基础+项目自我感知) | gui(cfg) | 最稳,前缀 |
| 2 | **工作记忆区(指针数据区)** | memory `ContextWindow` | 命中率高近乎不变 → 命中缓存 |
| 3 | **8K 最近记忆 ring** | memory(从 timeline 尾取) | 每轮变,有意不优化,只损自己 |
| 4 | 当前用户回合 | gui | 最易变,最末 |

## 铁律(决策日志逐字,不可违)

1. **信息两态**:上下文里信息只有"在/移除",**原位不动、不随缓存队列重排**;排队的是指针(在缓存,不在上下文)。已在上下文的不重复拼。
2. **工作区非线性**:载入顺序≠时间顺序 → **每块带完整时间戳**;提示词明示"判先后按时间戳,不按本区位置"。
3. **8K ring 永远放最末 + 着重特殊标记**"这是最近记忆";固定大小循环覆盖最旧。
4. **每区独特标记 + 格式**(不只一行标签):醒目边界 + 区内角色说明,持续点明这块是什么。
5. **尺寸随模型可调**:工作区与 ring 大小按模型上下文窗口成比例,默认可推算 + 用户可改,留剩余上下文余量。
6. 原"活跃线性记忆"已取消(冗余 + 破缓存)。

## 子阶段

- **P4a 区块模型 + 两态工作集 + ring + 时间戳(首个可用切面)**[本次]
  `Region/ContextBlock/ContextWindow`;`Memory::assemble_context(query,sub)` 跑 retrieve→按指针/命中调入工作集(两态去重)+ 尾部 ring;gui 替换 `agent.rs` 朴素组装,套区标记。memory 单测 + gui 测试绿。
- **P4b 预算淘汰 + 稳定前缀**:字符/token 预算,超额淘汰最冷(append-only 保前缀稳定);两态严格。
- **P4c ring 定型**:8K 固定预算,永远最末,特殊样式;与工作区去重(ring 覆盖近因,工作区排除 ring 内 id)。
- **P4d 尺寸随模型可调**:Settings 加工作区/ring 预算,按模型窗口推算默认,留余量;UI 可调。
- **P4e 原文三层置换 + 提示词收尾**:永久/冷归档/活跃的换出落盘(冷归档),区标记/角色说明打磨。
- **并入的债(用户:依赖 P4 的 BUG 并进 P4)**:组装层落地后,`get_status`/`get_memory_stats` 的
  `pointer_count`(`store.edge_count()`)、三级缓存命中率(`cache_stats()`)随手接真;工作区常驻数也可观测。

## 验证

每子阶段 `cargo test --workspace` 全绿;关键真机:组装出的 messages 顺序稳定(同输入跨回合前缀字节一致=可缓存)、工作区两态不重复、ring 在最末且带标记、每块有时间戳。

## 已知限制(自审 2026-06-01,夯实 P4 时记录)

1. **预算分区独立、非整窗感知**:`working_budget_chars` 只卡工作区自身,未把 system prompt + ring + 工具定义 + 当前回合算进模型总上下文窗口。用户把工作区设过大可能撑爆模型窗口。当前缓解 = `suggest_working_chars` 默认留 35% 余量;**真正的整窗预算编排(已知 system/tools 实际占用后回扣)留作后续**(需 tokenizer 或更准的占用估计)。
2. **预算单位是 UTF-8 字节(`content.len()`),非精确 token**:作为 token 的保守上界代理(中文 3 字节/字,按字节卡会更早淘汰,偏安全)。精确 token 计数留后续。
3. **中段淘汰会打碎该点之后的 prompt 缓存**:淘汰最冷若发生在工作区中部,其后字节偏移变化。命中率高时工作区近乎不变、淘汰罕见,可接受;已用测试 `working_region_is_byte_stable_across_turns` 锁住"无淘汰时逐字稳定"这一主路径。
4. **被 supersede 的节点可能在工作集残留**陈旧内容,直到被预算淘汰(append-only 固有现象,影响小;被取代的节点通常不会再被检索命中)。
