# 被造物 — 自由展示与交互(造物 UI 半)工程设计

> 原理层:`设计/00-交互层` 推论 10(被造物层 / 可视化优先 / 造物感知 / 延伸≠另一个程序);`设计/05-工具系统` 推论 8(造物执行器 + 交互独立通道 + 覆盖层);`设计/02-记忆检索` 备注(造物交互的记忆:默认丢+主动留);`设计/06-多Agent协作`(过程半复用)。
> 架构公理:`系统架构/00`(一切能力皆执行器、一个注册表一条分发路径)。
> 复用基座:`计划/活的IDE-UI执行器.md`(已落地的"声明式 surface + 单一分发 + 往返 ack + perceive";本计划是它的一次泛化)。
> 用户决策逐字:`用户决策/决策日志.md` 顶部「造物」三条(范围收敛 / 轮询=主动推送+覆盖层 / 外部输入 A·B 组)。
> 状态:**工程层设计稿,未实现。** 基质已定 = A(沙箱 HTML)。

## 一句话

把"活的 IDE"从"操控**预置**面板"泛化到"操控**AI 现造**的沙箱 UI"。**过程半(计算/定时/自激活)已有,本计划只补 UI 半。** 链路 90% 复用活的 IDE,真正新增的就三样:沙箱渲染目标、持续的交互回传通道、覆盖层。

## 目标

让 AI 能**现造一块 UI**(自由 HTML:表单/图表/棋盘)、**双向操控并感知**它——展示给用户看、收用户的点击/输入、主动往里推消息——而**无需为它另建后台**:回调穿回同一条 Agent 循环,AI 本身就是后台。对手/智力就是 GrowBox 本体那颗大脑(延伸≠另一个程序,`00` 推论 10)。

## 第一性原理:这是泛化,不是新系统

1. **不新建脊柱**:`render_artifact` 是又一个普通执行器,经 `registry.dispatch` 唯一分发(像 `ui_control`)。零特判、零新 Dispatch 变体。
2. **过程半复用**:造物的计算/定时/端口自激活 = 现有 `spawn_task` + `DoneWhen`(`tasks.rs`)+ Supervisor 唤醒(`supervisor.rs`)。本计划不碰。
3. **不撒谎**:渲染/操控的"成功"必须前端**往返验证**回报(复用 `EventSink::ui_round_trip` / `UiAckRegistry`),否则飞轮学进假结论。
4. **守沙箱**:造物内容跑在 iframe sandbox;**回调只回传数据给 AI,不能直接触发文件/shell 执行器**(那些仍走安全门 `05` 推论 5)。覆盖层是 GrowBox 可信 chrome,浮在沙箱之上,非真注入不可信 DOM。
5. **宿主层不沾**:实时输入/退避/覆盖层只活在被造物,GrowBox 本体聊天不沾(`00` 推论 10)。

## 核心模型:造物画布 = 两层 + 三条消息流

```
┌─────────────────────────── 造物画布(新 surface)───────────────────────────┐
│  覆盖层 (GrowBox 可信 chrome,右上角槽)         ← [流3] 主动推送(下行)      │
│  ┌────────────────────────────────────────────┐                            │
│  │  内容层 (iframe sandbox,AI 现造 HTML)        │ ← [流1] render(下行)       │
│  │   表单 / 图表 / 棋盘 + 操作接口(回调)        │ → [流2] 交互回传(上行)     │
│  └────────────────────────────────────────────┘                            │
└────────────────────────────────────────────────────────────────────────────┘
```

| 消息流 | 方向 | 性质 | 复用 / 新增 |
|---|---|---|---|
| **流1 渲染** | AI→画布 | `render_artifact(html)`,往返拿验证态 | 复用 `ui_round_trip` |
| **流2 交互回传** | 画布→AI | 点击/输入 → **造物交互通道**:不进聊天记录、有界瞬态、显示在画布 | **新增**(持续事件流,非一次性 ack) |
| **流3 主动推送** | AI→覆盖层 | 提示/拿主意,走双受众(显示+perceive 永久) | 半复用(意图下行)+ 新覆盖层 surface |

## 三个面,别混(用户 2026-06-03 厘清)

- **造物内容层**(AI 现造的 UI:表单 / 棋盘 / 图表 / 文件夹树)→ **一律沙箱 iframe**(AI 写的 HTML 不可信,正儿八经的展示也不例外)。交互造物挂回调 + 消息通道 + 自检(流2/3);**纯展示造物只渲不挂**(流1,无流2/3/自检)——它是"无交互的造物",不是免沙箱轻路。
- **造物覆盖层 = AI 的"吐槽 / 建议"槽**(右上角,给用户提建议、拿主意、吐槽的地方)→ **可信 SolidJS 浮层**(决策2),内容是 AI 的话、固定展示,**不进沙箱**(它是 GrowBox chrome,不是造物内容)。**★用户原话"展示直接浮层就好了、固定展示"指的就是这块,不是展示造物。★**
- **聊天框**(宿主层)不在此列。

## Phase 划分(每相一个可验证里程碑;相间按 [[phase-pacing-cooldown]] 冷却)

### Phase 1 — 展示半(流1)
- **后端**:`executors/artifact.rs` 的 `render_artifact(html)` 执行器。`risk()=Safe`;`execute()` 走 `ctx.ui.round_trip(UiIntent{action:"render_artifact", payload:html, await_ack:true})` → 拿 `UiAck{applied,state}` → 返回验证态(沙箱里渲成了/失败原因)。无前端(测试)时诚实失败。
- **前端**:新 surface `artifact-canvas` 进 `ui-actions.ts` 的 `PANELS` 注册表 + `register_ui_surfaces` 上报后端。收 render 意图 → 把 html 灌进 **sandboxed iframe**(`sandbox` 属性关网络/存储/顶层导航)→ 回 `ui_action_ack` 带渲染态。
- **验证**:AI 渲一个静态展示(文件夹结构 / 数据图表),拿到验证态文本。纯展示、无交互。

### Phase 2 — 交互半·离散(流2)
- **沙箱通信桥**:iframe ↔ GrowBox 用 `postMessage`。约定:造物 HTML 里交互元素带回调标识(`data-gx-callback="<id>"` 或暴露 `gx.emit(id,value)`),触发即 postMessage 出来。
- **前端**:桥收到 → emit `artifact_event(canvas_id, callback_id, value)` 给后端。**不走 chat-chunk、不进聊天气泡**。
- **后端·新通道**:造物交互**不进对话历史**,进**有界瞬态环**(复用/仿 `perceive_transient`,ring-only;容量挂已有可设的 `TransientCapsConfig`,见 0-OPUS20 G 组)。下一回合 `assemble_context` 把**当前造物态**(有界、最近 N 条)喂给 LLM,LLM 反应 → 可再 `render_artifact` 更新画布(闭环)。
- **验证**:五子棋 — 用户落子 → AI 收到(不进聊天)→ 回应并更新棋盘。**对手=本体大脑**,不接额外 AI。

### Phase 3 — 主动推送 + 覆盖层 + 实时输入(流3)
- **覆盖层 surface**:`artifact-overlay`(GrowBox 可信 chrome,绝对定位浮在 iframe 上,右上角槽)。`push_artifact_notice(text)` 意图 → 渲进覆盖层(不进 iframe、不进聊天)。
- **实时输入采样**:无提交键输入框 → 前端按"**内容是否变化**"做**指数退避**采样(变化才 emit、无变化就睡)。AI 收到要意识到"**可能没输完**":若要据此**行动**先问用户是否输完;纯对话不问(`00` 推论 10,非必要不读取)。
- **双受众**:AI 的主动介入 = 它的决定+言行 → **显示**在覆盖层 + **perceive 进永久记忆**(`感知告知-双受众` 模式,但显示面是覆盖层非聊天)。
- **验证**:模拟 AI 邮箱,输地址犹豫 → AI 在右上角弹提示/替他拿主意。

### Phase 4 — 记忆收口(默认丢 + 主动留)
- **`node_kind` 加受控 kind `ARTIFACT`**(造物;`label` 双语)+ 进 `controlled()` + 完整性守卫(`02` 备注 / `node_kind.rs` 模式)。
- **默认丢**:交互流水留在有界瞬态环,满了自动滚,不落时间线(省 token,不花判断)。
- **主动留**:只有 AI 判定"值得永久记的结论" → `perceive` 落时间线打 `artifact` kind + **自描述 content**(哪个造物、当时情形、虚拟时间/名字)。content 被 embedding → 语义检索 + 按 kind 筛。
- **造物关闭/毕业**:可触发一次摘要,决定落不落记忆。"造物日志"查看机制**后补**(标签先行零迁移)。

### Phase 5 — 过程半对接 + 收尾(多为复用/可选)
- **过程半**:重造物(服务/监控/常驻定时)挂现有 `spawn_task`+端口自激活(`06`);本计划不新增。
- **定时器糖(可选)**:`set_timer(delay, reason)` 薄执行器 = 包 `spawn(sleep)` + reason 进 label,唤醒消息读"定时器到点:<reason>"。**非新能力,可做可不做**。
- **毕业(创世级,不在本计划)**:延伸态 → 独立程序见 `创世级更新计划/内核随行-被造物毕业为独立程序.md`(待讨论)。

## 横切铁律(贯穿各相)

1. **守沙箱**:内容层 iframe sandbox;覆盖层是 GrowBox 可信层非真注入。
2. **回调不绕安全门**:造物回调只回传数据,要碰文件/shell 仍走 `05` 推论 5。
3. **不撒谎**:渲染/操控成功 = 前端验证回报,非"已发出请求"。
4. **不进聊天记录**:造物交互(流2)与主动推送(流3)都不进对话历史,各有其面(画布 / 覆盖层)。
5. **默认丢 + 主动留**:瞬态省 token,永久靠 AI 主动 perceive。
6. **造物自检关(感知自测)**:交互造物在 `finish` 前**必须自测全部感知点**(见下专节),不自测不许结束——"不撒谎"从单次 render ack 泛化到整个交互面的感知闭环。

## 造物自检关(感知自测,强制 finish 前置)

> **设计原话(用户 2026-06-03)**:"必须有自我调试功能,自己预计哪些可以感知,完成之后发送指令(自己想办法设计)测试,保证自己的各个感知正常,才能结束任务。"

交互造物建好后,AI **不得直接宣布完成**,必须走一遍感知自测:
1. **预计感知点**:AI 列出自己**预期能感知**的每一条(每个 `data-gx-callback` 回调、每个输入、实时采样、主动推送回执)。
2. **平台自测原语**(本计划提供):因回调是**声明式**的,GrowBox 能**自动枚举造物的全部回调面**,逐个**合成触发**(向每个声明元素派发合成事件),回报"对应 `artifact_event` 是否到达 / AI 是否真感知"——一张 checklist。
3. **AI 自己设计测试**(用户:"自己想办法设计"):测哪些、怎么判读、预期与实际是否一致,由 AI 决定;平台只给"合成触发 + 回报到达"这一原语。
4. **全绿才放行**:任一感知点没打通 → 任务**不算完**,AI 据失败项修造物(改回调标识 / 补接线)再测,直到各感知正常方可 `finish`。

> 本质 = 治"造物版虚假成功":AI 不能造完棋盘就说"能玩了",得先证明"我真能感知到落子"。声明式回调让这步**大半可自动枚举**,是选声明式的额外红利。

## 关键技术决策(留作编写时拍,先列不预设死)

- **沙箱通信桥协议**:`postMessage` 的消息格式(渲染下行 / 事件上行 / 推送下行三类的 envelope)。
- **回调标识约定 → 已定:声明式优先 + 逃生口**。主路 `data-gx-callback` / `-value` / `-realtime`(事件委托捕获),逃生口 `gx.emit(id,payload)`(算/聚合复杂值时用);两者同一注入引导 + 同一 postMessage envelope。**额外红利**:声明式 → GrowBox 可自动枚举回调面 → 利造物自检。
- **有界瞬态容量**:复用 `TransientCapsConfig` 既有旋钮,还是给造物单开一个(一局棋的事件量 vs 其它瞬态)。
- **覆盖层实现 → 已定:SolidJS 绝对定位层叠**(可信层浮在沙箱之上 = 结构性守沙箱;同进程零 IPC;iframe 内容换、浮层不换)。展示轻路也复用这层。
- **多造物并存**:`canvas_id` 区分;是否允许同时开多个造物画布(MVP 可先限一个)。
