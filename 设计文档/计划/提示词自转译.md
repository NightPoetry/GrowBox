# 提示词自转译(self-transpile prompts)— 已实现(v1)

> 状态:**v1 已落地(2026-06-08,git feature/artifact)**。下方规格草案保留作背景;实际 as-built 见文末「落地(as-built v1)」。
> **原理归属**:这是内核原理「自我负责」的**输入侧**(`设计/08-自我负责.md` 推论2);
> 与已实现的**输出侧**「主动自检」(推论1)同源。理论锚点 = decoder 自亲和(模型最能执行自己写出来的话)。

## 一句话

用**当前加载的主模型**把所有"喂给模型的提示词"按它自己的风格重写一遍,让它更清楚、更易被该模型执行;
凡是喂给**非主模型**(潜意识/判定/蒸馏模型)的那部分提示词,由**那个模型自己**转译自己那份。

## 理论依据(用户给出)

纯 decoder 模型:它的**输入注意力的向量空间与模式**,与它**自己输出**的高度相似。
所以"由模型 A 重写、再喂给模型 A 执行"的提示词,A 理解/执行得最好——让每个模型读自己的话。
推论:提示词不该是"人写的、对所有模型通用的一份",而应**按当前模型自转译**成它的母语风格。

## 范围:哪些是"模型可见提示词"(转译对象)

需先做一份清单(实现前核 file:line)。当前已知的模型可见提示词来源:

- **主模型可见**(由主模型转译):
  - 系统提示词:`prompts/agent/system.{zh,en}.md`
  - 工具说明:`prompts/tools.i18n.json` 的 `llm_desc`(zh/en)+ 各参数 `description`
  - 工作流节点引导:内置工作流(`workflow_store.rs` builtin_*)+ 持久工作流(redb/造物文件夹)的 `node.prompt` / `description`
  - 脊柱注入的脚手架(`agent/render.rs`):流程召回块、可执行流程块、节点引导、自检指令(`self_verify_prompt`)、空转中性引导等
  - process 配方("照做"块)本身是记忆内容,**不转译**(那是数据,不是提示词)
- **非主模型可见**(由对应模型各自转译):
  - 潜意识/判定模型:`judge_relevant`、`distill`、记忆重排/摘要/做梦相关提示词(在 `growbox-learn` / `growbox-llm` 的对应实现)
  - 嵌入模型:无自然语言提示词(跳过)

## 实现要点(工程草案)

1. **不覆盖原文,做"按模型分桶的转译覆盖层"**:原提示词是真理来源(source);转译产物按
   `(模型 id, prompt_lang)` 存一份覆盖(redb kv / 文件),运行时优先用覆盖、缺则回退原文。
   换模型 / 重新转译 干净;原文永不丢。
2. **谁转译谁**:主模型转主模型可见的;判定/蒸馏用其对应模型(潜意识槽)转自己那份。
3. **触发**:设置里一个动作("用当前模型重写提示词")/ 或 idle 时后台做。成本高(每条提示词一次 LLM 调用)→
   要进度提示 + 可中断;不该每次启动都跑。
4. **保真校验(与主动自检同源)**:转译易丢关键指令(语义漂移)→ 转译后过一道校验
   ("改写后是否保留了原指令的全部约束?"),不通过则保留原文 / 标记。可复用 `self_verify` 的思路。
5. **缓存影响**:转译后的提示词成为新的 byte-stable 前缀——只要"每模型一份且稳定"就不破坏 deepseek KV 缓存
   (见 [[deepseek-cache-and-reasoning]]);转译动作本身会使旧缓存失效一次(可接受)。
6. **i18n**:`prompt_lang` 为 zh/en,按语言各转译一份;界面四国与此无关(转译只针对"喂给模型"的提示词)。
7. **可关 / 可还原**:开关 + 一键还原到原文(同主动自检的"可关省 token"哲学)。

## 接原理

贴 GrowBox "一切数值/行为可设"(转译开关 + 触发)+ "自我感知/自我改进"线;与 [[主动自检-grounded-verification]]
同源(都是让模型对自己的产物负责)。是"活的 IDE / 越用越强"的一环:模型把工具/流程的说明改写成自己最顺的话。

## 风险

- 语义漂移丢约束(靠保真校验 + 保留原文兜底)。
- 成本(很多次 LLM 调用 → idle 做 / 按需触发)。
- 多模型各转各份的工程复杂度(分桶存储 + 回退链要清晰)。

## 落地(as-built v1,2026-06-08)

**核心**:`crates/growbox-gui/src/transpile.rs` —— 进程级单例(`enabled` + 两个模型 id + 覆盖表),
`connect` 经 `configure()` 推入(覆盖表从 redb kv `prompt_transpile_overrides` 载入)。
- **角色 tag**:`PromptRole::{Main, Subconscious}`,决定用哪个模型转、覆盖落哪个桶。今天潜意识 == 主模型
  (同一个 `Settings.model`),故二者解析到同一 id、同桶;**将来拆独立潜意识模型时,`okey(model,lang,key)` 按
  不同 model 自动分桶,转译层一行不改** —— 这正是"谁转译谁"用 key 编码、不需要现在动架构的设计兑现。
- **唯一取用 chokepoint** `tr(key, role, lang, original)`:关 / 模型空 / 该桶无覆盖 → 逐字原文(**零行为变更**)。
- **静态目录** `CATALOG`(zh+en):把内联提示词收编 —— `agent.self_verify`(含 `{summary}` 占位符)+
  `subconscious.judge_relevant` / `judge_edge` / `distill`。zh 文本与接线前硬编码**逐字一致**(单测护栏
  `catalog_zh_matches_legacy_hardcoded_byte_for_byte`,保默认中文用户零变更)。
- **动态来源**:工具 `llm_desc`(`registry::tool_desc_units` + 取用处 `tr`,key=`tool.<name>.llm_desc`,
  保留 `{catalog}` 占位)+ 系统提示 `agent.system`(`cmds/chat.rs` 取用处 `tr`;只转静态那半,项目上下文是
  运行时数据不转)。
- **接线点**:`bridge.rs`(judge/distill/judge_edge,新增 `with_prompt_lang`)、`cmds/chat.rs`(系统提示)、
  `registry.rs`(工具说明 x2)、`agent/render.rs::self_verify_prompt`(改取 catalog + 替换 `{summary}`)。
- **重写动作** `cmds/transpile.rs::transpile_prompts`:**不持锁跑 LLM**(先加锁快速收集单元→放锁后逐条调模型);
  每条 = 重写(`rewrite_instruction`)→ 机械保真 `fidelity_ok`(占位符存活 + 长度 0.25~4x)→ 语义复核
  (问同模型有没有漏约束/丢占位符,best-effort)→ 过则写覆盖。覆盖**合并**进既有表(别的模型旧覆盖保留)、
  落 redb + 刷新内存。进度走 `transpile-progress` 事件。
- **开关/状态命令**:`set_transpile_enabled`(即时翻 + 落库)、`get_transpile_status`(开关+覆盖条数+模型)。
- **前端**:`Settings → AgentTab`「提示词自转译」区(开关 + 「用当前模型重写」按钮 + 进度/覆盖条数/上次结果);
  `core::Settings.prompt_transpile`(默认 false);i18n 四语(`transpile*` 键)。
- **测试**:transpile 模块 11 单测(开关短路/命中覆盖/角色分桶/占位符提取/保真校验/zh 逐字护栏;
  全局单例用测试锁串行化防竞态)。全工作区 357 绿 / clippy 零 / 前端 tsc+vite 干净。

## v1.2:UI 选区化 + 并发 + 缓存暖机(2026-06-08,用户反馈)

**UI 改版**(原"开关 + 按钮 + 列表"手感差):去掉「启用」开关 —— 启用与否由「**当前提示词**」选区决定:
选「默认(原文)」= 不转译(覆盖空);选某个历史包 = 启用该包。运行时 `enabled = !active_overrides.is_empty()`
(connect/activate/import/delete 都按此设),不再有独立 bool 开关。重写按钮配图标、与并发输入同排;历史版本
管理(改名/导出/删除)折叠在选区下方。

**并发转译**:`Settings.transpile_concurrency`(默认 4,可设)。`transpile_prompts` 用 tokio `Semaphore`+`JoinSet`
限并发跑各单元(每单元 = 重写 + 语义复核两次 LLM 调用)。每单元彼此独立(冷请求),并发是降低墙钟的主手段。

**★Deepseek 缓存特性:暖机后并发(用户洞察)★**:每条转译的**常量 system 前缀**(`rewrite_system` 重写指令 /
语义复核指令,各 zh/en 一份)第一次发才被 Deepseek 缓存住。若一上来就并发,头一批会在前缀尚未缓存时同时发出
→ 各 miss 一次。故 `transpile_prompts` **先每个语言各取 1 条串行跑**(zh+en,坐实两份 system 前缀进缓存),
再放并发 → 余下请求全命中前缀(只算"原文 + 输出"的新 token)。**这是 Deepseek 专享的 KV 前缀缓存特性;对无前缀
缓存的模型只是把 2 条放前面串行、无害。** 另:把固定指令做成 system(原文放 user)本身就是为最大化这段前缀的命中。
(转译本身固有是一批"冷"请求 —— 每条重写不同提示词,跨请求只有 system 前缀可共享;别指望它"缓存很热"。)

## 历史提示词版本库(可后悔,2026-06-08 用户要求)

重写**必须可后悔**:每次「用当前模型重写」不是覆盖、而是存成一个**新版本**,可在 UI 列史/加载/改名/删除。
- **存储** `crates/growbox-gui/src/transpile_store.rs`:整库一条 redb kv `transpile_library`;每版 =
  覆盖表 **gzip + base64 压缩串** + meta(id/name/model/count/created)。核心是 `Library` 上的**纯方法**
  (脱库单测),`Store` 只是 load→改→save 薄包装。「默认(原文)」是隐式版本(id=`default`,空覆盖=用原文),
  **永不可删、是兜底**;序号单调只增不回收(防与已删版重名);旧 v1 单覆盖表首次载入自动迁移成一个版本。
- **行为**:重写 = 合并(当前激活版 + 本次重写)→ `add_version` 存新版并置激活(旧版全留着,可后悔)。
  激活某版即时把其覆盖表推入全局取用层;激活 `default` = 还原原文;删激活版 → 回落 default。
- **命令** `cmds/transpile.rs`:`transpile_list_snapshots` / `transpile_activate_snapshot` /
  `transpile_rename_snapshot` / `transpile_delete_snapshot`;`connect` 改读 `transpile_store::active_overrides`。
- **UI**:AgentTab 自我负责区「历史提示词」列表(默认原文置顶删不掉 + 各版本名可改/显示模型·条数/加载/删除);
  四语 i18n(`transpileHistory`/`transpileLoad`/...)。
- 测试:`transpile_store` 6 单测(pack/unpack 往返 + 坏串安全回落、加版置激活+序号命名、激活回退+还原 default、
  改名护默认、删除护默认+回落、序号不回收)。全工作区 363 绿。
