# AI 记忆置换系统(总纲)

> 体系命名(用户 2026-05-31,原话见 `用户决策/决策日志.md`):GrowBox 的记忆架构本质是一套 **AI 记忆置换系统**——类比操作系统的虚拟内存/分页。
> **本质机制 = 内存置换**(把无界记忆映射进有限的 LLM 上下文窗口);**缓存只是加速,不是本质**。

## 一句话
记忆无界、上下文有限。置换系统负责:把"现在该看的"那一小块从磁盘换进上下文窗口,把"暂时不看的"换出冷归档;指针网是地址映射(页表),让你不必全量扫就能定位该换入谁。

## 与操作系统虚拟内存的对应
| 操作系统 | GrowBox 记忆置换系统 |
|---|---|
| 虚拟地址空间(无界) | 全部记忆(磁盘 redb,无界增长) |
| 物理内存 RAM(有限) | LLM 上下文窗口(活跃工作集) |
| 页表 / 地址翻译 | **指针网**(node→node 边,按 source 邻接) |
| 缺页 → 从 swap 调页 | 检索 miss → 沿指针从冷归档/磁盘按需召回 |
| 页面置换(LRU/clock) | **指针数据区(工作记忆区)** 按指针 LRU 调入/淘汰(三级缓存热度决定换入谁、淘汰谁) |
| swap / 磁盘 | **冷归档**(redb) |
| TLB / page cache(加速,非本质) | **三级缓存**(热指针 1:2:4,在 RAM、不在上下文) |

## 上下文窗口分层(决定 prompt 缓存命中 = 省钱)
顺序从稳定到易变(稳定前缀在前 → LLM prompt 缓存可复用):
1. **system prompt**(最稳)。
2. **指针数据区(工作记忆区)**:三级缓存里热指针所指向的**数据**载入此区,是**模型回答的直接参考**。**信息只有"在/移除"两态、原位不动、不随队列重排**;排队/LRU/提队首/淘汰操作的是**指针(元数据,在上下文之外)**,不是信息。命中率高 → 此区几乎不变 → 缓存命中。已在上下文的信息**不重复拼**。
   - **大小用户可调 / 随模型上下文窗口放大**(大模型配更大此区,驻更多数据)。
   - **非线性区**:载入(摆放)顺序 ≠ 真实时间顺序,故每条载入消息**带完整时间戳**;提示词须明示"判断历史先后**按时间戳,不按它在本区的前后位置**"。
3. **8K 最近记忆**(固定大小、循环使用 ring buffer):**永远放上下文最末**,**着重特殊样式标记"这是最近记忆"**。每轮都变,但只 8K → **有意不为缓存优化**,放最末只损它自己这点,不打乱前面稳定前缀。
4. **当前用户回合**。

> 原"活跃线性记忆(128K 线性原文 1/3 刷新)"分区**已取消**(见决策日志):与指针数据区按需调入的数据重复,且其滑动刷新打碎 prompt 缓存。原文的近因由 8K 最近记忆覆盖,相关历史由指针数据区按指针调入。

**两条铁律**:
- **信息两态**:上下文里的信息只有"保存 / 移除",不随缓存队列移动;队列里排的是**指针**不是信息。常驻前缀因此稳定 → 省 prompt 缓存钱。
- **每区独特标记 + 格式**(不只一行标签):每个区用醒目边界 + 区内角色说明**持续**点明"这块是什么",光标签 LLM 易忘。8K 最近记忆尤其着重标记。

**尺寸随模型可调**:指针数据区与 8K 最近记忆的大小非定值,按模型上下文窗口成比例配置(1M 模型就放大指针数据区),按窗口推算默认值、允许用户调,并给"模型剩余上下文"留余量。

> 注:三级缓存 = **指针(元数据)缓存**,在 RAM、不在上下文;指针数据区(工作记忆区)= 这些热指针在上下文里对应的**数据**。指针在缓存里排队,数据在上下文里稳定驻留。

## 三层记忆(置换的对象)
- **永久**:全部原文/结论,真相在磁盘 redb,无界。
- **冷归档**:换出上下文、但仍在盘上,经指针按需召回(= swapped out)。
- **活跃**:当前在上下文窗口里的工作集(= resident)。

## 落点与边界(给实现者)
- **核心是置换/页表,不是缓存**。先把"指针网=地址映射 + 冷归档=swap + 活跃窗口=resident + 置换策略"做对;三级缓存/64K 队列是加速层,后加。别把缓存当核心。
- 指针网必须**磁盘原生**(按 source 落 redb,一次读一个邻域),否则"页表"本身就装不下,见 `计划/precision-layer.md` 存储节。
- embedding(第一层 RAG)是"该换入谁"的粗筛入口之一,见 `计划/embedding-service.md`。
