# 跨平台迁移方案

> 用户决策 2026-06-01:**移除"纯 Rust 无原生依赖"红线,先保证 Mac 能用**;但**留好接口,Windows 等可再编译/迁移**。本文件记录:当前依赖现状、可换实现的 seam、磁盘原生升级路径、按平台迁移步骤。

## 现状:其实仍是纯 Rust,已可跨平台重编

红线虽松绑,但本会话的选型恰好**全是纯 Rust**,所以现在就能在 Mac/Win/Linux 直接 `cargo build` 重编,无运行时原生库要按平台分发:

| 用途 | 选型 | 原生依赖? |
|---|---|---|
| 嵌入运行时 | `candle` 0.9 | 无(0.10 会拉 onig 原生库,故钉 0.9) |
| 分词 | `tokenizers`(关 onig,走 fancy-regex) | 无 |
| 模型下载 | `hf-hub`(rustls) | 无 |
| 第一层向量索引 | `instant-distance`(HNSW,内存) | 无 |
| 图/KV 持久化 | `redb` | 无 |
| HTTP | `reqwest`(rustls-tls) | 无 |

→ 当前迁移到 Windows = 装 Rust + MSVC 工具链,`cargo build` 即可;模型文件平台无关;路径已用 `PathBuf`。

## 可换实现的 seam(留好的接口)

换实现不动调用方,两个关键 trait:

1. **`Embedder`**(`growbox-llm/src/embed.rs`):`LexicalEmbedder` / `RemoteEmbedder`(OpenAI 兼容)/ `LocalE5Embedder`(candle)。换模型或换本地/远程都在此切。
2. **`VectorIndex`**(`growbox-memory/src/index.rs`):`ArroyIndex`(arroy/LMDB,**磁盘原生 mmap,当前运行时默认**)/ `HnswIndex`(instant-distance,向量驻 RAM,无 store/测试用)/ `BruteForceIndex`(无依赖)/ 日后可接 `HannoyIndex`(见下)。换引擎不碰 `Memory::retrieve`。

## ★ 磁盘原生 ANN:已落地 = arroy(2026-06-01)

旧 `HnswIndex`(instant-distance)把**全部向量驻 RAM**——与"记忆置换系统(无界记忆不全驻内存)"thesis 不一致。已换成磁盘原生 ANN:

- **已用 `arroy` 0.6.4**(github meilisearch/arroy,Kerollmops):随机投影树 + **LMDB(heed 0.22)**,**mmap 支持比 RAM 大的索引**(正合 thesis)、并发只读。**选 arroy 而非更快的 hannoy = 用户硬规矩"依赖必须知名成熟+联网核实"**(arroy 0.6.x / 39 万下载 / 多年 Meilisearch 生产 / 版本干净;hannoy 仍 0.1.x)。同 Meilisearch 团队、同 LMDB 家族,`VectorIndex` trait seam 留着,日后 hannoy 成熟可 drop-in 换。详见决策日志 2026-06-01 + 记忆 `prefer-well-known-verified-deps`。
- LMDB(C,heed):vendored C 由 cc 构建时编进二进制,**各平台重编即可**(Tauri 本就需 cc/MSVC),无运行时原生库分发。

**已落地情况(`index.rs::ArroyIndex` + `memory.rs::open`)**:
1. 依赖 `arroy = "0.6.4"` + `heed = "0.22.1"` + `rand = "0.8"`;在 `index_dir/vector-index` 开一个 LMDB env(与 `growbox.redb` 并存,各管各:redb=节点/边/KV,LMDB=向量 ANN);map_size 1 GiB。
2. `ArroyIndex impl VectorIndex`:`rebuild`=clear + 逐条 add_item + builder.build(**当前全量重建**,O(N)每次开机/补向量);`search`=Reader.nns(k).by_vector,search_k 放大优先召回;Cosine 距离 d=(1-cos)/2 → 还原 cos=1-2d;ItemId(u32)↔node id 用内存 `Vec<String>`。
3. `Memory::open(store, index_dir)` 默认 `ArroyIndex`(打开失败 eprintln 回退内存 HNSW);`new()`/无 store/测试用 `HnswIndex`/`BruteForceIndex`(无需 LMDB env)。
4. 配合 P3d 时间线惰性化:向量在 arroy(LMDB/mmap),正文按 id 惰性从 redb 取 + 热尾缓存 → "无界记忆不全驻内存"两半合力达成。
5. **待优化**:`rebuild` 改增量 upsert/del(arroy 有 `add_item`/`del_item`),利用 LMDB 持久化免每次开机全量重建。坑:工具沙箱挡 crates.io(403 非病毒),拉依赖/编 LMDB-C 用 `dangerouslyDisableSandbox`。

## 按平台迁移步骤(checklist)

- [ ] Windows:装 Rust + VS Build Tools(MSVC,cc);`cargo build` 全工作区;`cargo tauri build` 出包。LMDB(若已上 hannoy)随之用 MSVC 编。
- [ ] Linux:Rust + gcc/clang;同上。
- [ ] 模型分发:见 `打包设计.md`(带模型包 / 不带模型包;resource→data→hf-hub 解析)。
- [ ] 路径/换行/大小写:已用 `PathBuf`;注意 Windows 大小写不敏感与长路径(必要时加 `\\?\` 前缀)。
- [ ] 原生目录选择器:`cmds.rs::pick_directory` 现走 macOS `osascript` —— **Windows/Linux 要换实现**(Tauri dialog 插件或平台命令)。这是目前唯一明确的平台专属代码,迁移时必改。
