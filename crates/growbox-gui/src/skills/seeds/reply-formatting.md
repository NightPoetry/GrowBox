# 技能:reply-formatting
触发:给用户写回复、且输出里包含链接、表格、列表、代码等需要正确渲染的格式时

你的回复经 Markdown 渲染给用户看。格式不规整,渲染就会错(链接点不开、表格散成竖线、列表挤成一行)。按下面几条写,渲染才正确、好读、可点。

1. **超链接**:凡是想让用户能点开的地址,写成完整可点形式,二选一:
   - Markdown 链接:`[发现梦想](http://localhost:3000/dream-board/)`(推荐,文字+地址分离最稳)。
   - 或裸的完整 URL,且**必须带 `http://` 或 `https://` 前缀**、**URL 后面留一个空格或换行**,绝不要让中文/标点紧贴 URL 末尾(写 `打开 http://localhost:3000/ 看看`,不要写 `打开http://localhost:3000/看看`——紧贴会把"看看"吞进链接、点开就坏)。
   - 一次列多个地址时,每个单独成行或用列表,别堆在一句里。

2. **表格**:用 GFM 表格,且**表格前后各空一行**(否则会被当普通文字、散成一堆竖线):

   ```
   (上一段正文)

   | 列1 | 列2 |
   |-----|-----|
   | a   | b   |

   (下一段正文)
   ```
   表头下面那行分隔符 `|---|---|` 必须有;每行列数和表头一致。

3. **列表**:每个条目**单独一行**、行首用 `- `(无序)或 `1. `(有序);别把多个条目用顿号/逗号堆在一行。列表前最好空一行。

4. **代码**:多行代码/命令/文件内容用三反引号围栏(```)包起来,可标语言(```js / ```bash);一行内的标识符/路径/命令用单反引号 `like_this`。别把代码裸贴在正文里。

5. **分块**:不同意思的段落之间空一行;标题用 `## 标题` 且前后空行。别把推理、命令、结果、结论全挤成一坨——分块才读得清。

要点:输出前想一眼"这段渲染出来长啥样"。链接能点、表格成形、列表分行、代码成块——这是对用户最基本的尊重。
