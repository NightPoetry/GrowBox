# 技能:ui-respects-attention
触发:设计/改动任何给用户看的界面(造物或 GrowBox 本体 UI)时

UI 的目标是**用户最小心智负担**:输入端少说、消费端少看、决策端少决定。据此:

1. **特殊语义内容必须有醒目、可区分的样式**(设计/00 推论8):工具调用、命令输出、等待、告警、内部消息——语义不同就样式不同,让用户一瞥即可分类、不必逐字读。把不同语义混成一片纯文本 = 把分类负担甩回用户的眼睛。
2. **失败/等待/加载必须有可见状态**(失败必可感知):任何异步操作(连接/保存/发命令)都要有 pending 指示 + 失败提示(走 notices 的 toast/感知双受众)。**绝不静默吞错**——用户点了没反馈、或失败了不告知,是本项目反复修的洞。
3. **留白不确定,别瞎猜**(预填可推断、留白不确定):能高把握推断的字段预填好,拿不准的留空——猜错=用户先发现再改(两步),留空=用户直接填(一步)。留空是诚实。
4. **一切数值可设**:凡影响行为的数值(容量/阈值/超时/上限)都给设置入口,有默认 + 有 UI 改的路径——藏死=替用户做了看不见的决定。
5. **可逆的直接做,不可逆的交还**:开关面板/滚动这类可逆操作 AI 直接做(零裁决);改用户数据/删除这类不可逆的,准备好但把最后一下交给用户确认。
6. **结论先行、可读优先**:展示信息第一句给结论;简短靠"少说不相关的",不靠压成黑话/箭头链。
7. **面向用户的文案走 i18n 四语**(zh-CN/en/ja/zh-TW),别硬编码(造物文件按其惯例可中文)。

要点:好 UI 替用户把"该做的那一下"做小或取消,并让每条信息一瞥可分类、每个失败都被看见。
