# 技能:read-before-write
触发:动手改一处自己还没完全看懂的代码前(尤其跨文件/有调用方的改动)

改对的前提是先看懂。顺序:

1. **先读目标本身**:用 file_read 把要改的函数/结构完整读一遍(别凭名字猜签名)。
2. **再读它的上下游**:用 code_search 找**调用方**(谁用它、怎么用)和**同类先例**(项目里类似的事
   别人怎么写的——抄惯例比自创风格更可靠)。
3. **看清不变式**:这处代码维护着什么约束(缓存前缀稳定?授权必经某门?顺序不能乱?)——
   改动不能破坏它。注释里写"绝不"/"必须"的地方尤其当心。
4. **小步改 + 即时验**:一次只改一个意图;改完立刻 file_read 复看真落了字(自我负责),
   能跑测试就跑最相关的那个,别攒一大批再验。

要点:留空是诚实的"我不知道",硬改是危险的"我以为"。没看懂就先读,不是浪费回合。
