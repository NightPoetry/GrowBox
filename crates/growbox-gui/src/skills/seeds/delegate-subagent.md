# 技能:delegate-subagent
触发:要把一块活外派给子 Agent(工作流分支调用)去做,决定给它多少上下文、多少工具、要它回什么时

派生子 Agent = 一次类型化函数调用,由三个旋钮定:看什么(上下文)/ 能做什么(工具)/ 返回什么(摘要还是全量)。先想清这三个再调,别默认就全量继承。

1. **先判值不值得外派**:会产生大量工具噪音(读一堆文件 / 反复编译 / 全库搜)、能并行、或有干净交付物 → 外派;否则就地做(派生本身有开销)。
2. **看它需不需要你此刻的全部背景(计划、已定决策),据此选 context_mode**:
   - **需要(是你工作的延续)→ `fork`**:继承你全量上下文(它什么都知道),但退出只把摘要交回、过程噪音不回灌你。给"需全背景的繁重机械活":大范围重构(配合 [[safe-refactor]])、反复编译验证。
   - **不需要(自洽查询,看一眼代码 + 一份简报就能答)→ `isolated`(调查员/审核员)**:只看你给的 input、退出只回摘要。给"查 X 在哪 / 审这段 diff / 核某断言";可并行扇出多个。
   - 轻量续做、分支工作就想留在你这里 → `inherit`(默认)。
3. **收窄它能用的工具**:调查员/审核员通常**只读**就够(给只读工具子集 = 它写不了、既安全又专注);fork 按活给全工具或所需子集。工具面由工作流节点的 tools 决定,节点外的工具它在执行闸门根本调不动。
4. **写清返回契约 return_spec**:要它带回什么、什么格式。默认只回最少充分信息(错误/警告/结论);极少数要原始全量才 `full=true`(慎用,会污染你的上下文)。
5. **给够简报**:isolated / 调查员看不到你的上下文,它需要的背景都得写进 `input`。**若发现要写巨长简报才够 —— 这就是该改 `fork` 的信号**(fork 本就继承你全部背景,不用你转述)。

要点:成本不对称,**默认调查员**(fork 每轮背全上下文,贵)。fork = "我要它知道一切、只别拿过程噪音烦我";调查员 = "我只要一个干净结论"。两者只差一个旋钮(看不看得到你),不是两套机制。
