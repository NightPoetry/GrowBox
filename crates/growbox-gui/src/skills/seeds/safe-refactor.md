# 技能:safe-refactor
触发:要重构(拆大文件/挪模块/改结构)但必须保证行为不变时

重构 = 改结构、不改行为。能不能安全重构,取决于你有没有"行为没变"的证据。

1. **先有安全网**:重构前确认有测试覆盖被动区(或先补)+ 编译器/类型在帮你(Rust 的类型 + `cargo check` 是强网)。没网就先织网,别裸奔改结构。
2. **小步、可逆、每步验绿**:一次只挪一件事(拆一个模块、提一个函数),每步 `cargo check`/相关测试绿了再下一步——别一口气重排十处再编译,红了无从定位。
3. **零行为变更优先**:纯搬迁(挪到子模块、`use super::*` 透传、pub use 重导出)时,逐字搬、不顺手改逻辑;想改逻辑就单独另起一次提交,别和重构混在一起(混了 review 不出哪是搬哪是改)。
4. **本项目先例**:god 文件拆分(agent.rs/memory.rs/cmds.rs/Settings.tsx 拆子模块)= 零行为变更范本——impl 跨文件、`use super::*` 透传、命令 `pub use` 重导出、测试迁出但逐字不动。skills.rs→skills/mod.rs+seeds/*.md 同理。
5. **验证等价**:重构前后跑同一组测试都绿 + 关键路径真机走一遍(见 [[verify-by-running]]);别只信"我只是挪了位置"。

要点:重构的纪律是"结构变、行为可证不变"。把"搬迁"和"改逻辑"分成两次,各自可 review、可回退。
