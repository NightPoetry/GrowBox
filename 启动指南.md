# GrowBox 启动指南

> 这份是**自包含的入口**:任何人(或任何新接手的 AI 会话)从这里开始,不需要先认识谁。
> 最后更新:2026-06-01(P5 做梦/睡眠/疲劳 + 潜意识仲裁器落地后)。

## 1. 这是什么

GrowBox = 一个**越用越强的持久学习型 AI 编程助手**(Tauri 桌面应用)。
后端 Rust(6 个 crate),前端 **SolidJS**(不是 React)。核心特色是"记忆置换系统"——把无界记忆映射进有限的 LLM 上下文窗口(类比操作系统的虚拟内存/分页),并有一套"飞轮"把每次实践沉淀成可复用的知识。

## 2. 一分钟跑起来

环境:macOS、Rust(edition 2021,装最新 stable 即可)、Node(给前端)。

```bash
# 跑全部测试(最快的"它还活着吗"验证)
cargo test --workspace

# 真机验证与 LLM 的通信(需要 DeepSeek API key,自己的,别写进文件)
DEEPSEEK_API_KEY=<你的key> cargo test -p growbox-llm --test live_deepseek -- --ignored --nocapture

# 启动 App(开发态)
scripts/launch.sh
```

改了前端后必须先手动重建(`tauri.conf.json` 的 `beforeBuildCommand` 是空的,不会自动重建):

```bash
cd crates/growbox-gui/frontend && npm install && npm run build
```

## 3. 出包

```bash
scripts/build-official.sh   # 正式包:无调试桥、无 :19999 端口、无 debug IPC
scripts/build-test.sh       # 测试包:带 window.__GROWBOX__ + :19999 + debug_eval/e2e_report
# 末尾可透传 cargo 参数,如:scripts/build-official.sh --debug
```

二进制名是 `growbox`。脚本已内置前端重建步骤。

## 4. 接手开发?按这个顺序读(真理来源)

1. **`交接报告.md` 顶部的 `0-OPUS` 横幅**(最新那条 = 当前进度 + 续点 + 锁定决策)。下方旧的 DeepSeek 描述可能过时,以最新横幅为准。
2. **`设计文档/AI记忆快照/`**:这是负责开发的 AI 会话的**记忆副本**(新会话没有外部记忆,靠这里恢复)。先读它的 `MEMORY.md` 索引,再读标 ★先读·当前活跃★ 的 `precision-layer-progress.md`。
3. **`设计文档/`= 设计真理来源**:
   - `记忆置换系统-总纲.md`(记忆架构总纲)
   - `用户决策/决策日志.md`(用户原话逐字,冲突以原话为准)
   - `设计/`(原理层)、`系统架构/`(工程层)、`具体系统设计/`(各系统 as-built + 用什么库)

4. **`CLAUDE.md`**:给 AI 的工作手册(铁律 + 构建坑)。

## 5. 铁律(不可违反)

1. 除前端交互设计外,**所有非 UI 结构全部重写**;旧代码不作实现依据。
2. CLI 已砍,只留 GUI。
3. **有疑问自己做实验验证**,不继承旧 AI 的未验证结论。
4. 用户要能跑的产品,不懂 Rust 细节;技术决策自己定 + 自测。
5. **禁止 Emoji**(代码/文档/提示词皆是)。
6. 只在「要 key / 要授权 / 要看成果」时找用户;中途别频繁打断。
7. **永不触碰** `/Volumes/UserData/Projects/Helper/GrowBox_DeepseekV4Byopus/`(只读参考)。
8. **选依赖必须知名成熟 + 联网核实**(下载量/谁在用),不靠记忆拍板。
9. 用户每做决定性设计,**逐字**记进 `设计文档/用户决策/决策日志.md`。
10. 文档/记忆**非增量改动前,先把原文备份到 `History/`**(备注改动 + 日期)。
11. **API key 绝不写进任何文件**——每次问用户要。

## 6. 给用户的暗号:`置换归位`

在**新会话**里对 AI 说 **`置换归位`**(或近似说法),AI 会按 `设计文档/AI记忆快照/resume-passphrase.md` 的指引,把全部上下文调起、报一句"已归位",再从当前续点继续。
这是给 AI 的"开机咒语",省去你每次重新讲一遍项目。
